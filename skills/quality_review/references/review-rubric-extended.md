# 质量审核 · 8 维评分扩展 Rubric

> SKILL.md §2 的详细打分标准。每维度 10 分制，细节如下。

---

## 1. 事实准确性（factAccuracy）· 权重 20%

| 分数 | 表现 |
|------|------|
| 9-10 | 所有具体数字 / 人名 / 时间 / 地点 / 引言 100% 经得起公开核查，附来源链接 |
| 7-8 | 主要事实准确，1-2 处可疑（但未经核查） |
| 5-6 | 关键事实无来源；或 1 处错误（数字差异 < 10%） |
| 3-4 | 2+ 处错误，或关键人名时间错 |
| 0-2 | 大段捏造；或政治敏感事实错误 |

### 评分示例

- **案例 A**：稿件提到「深圳 AI 产业新政设立 200 亿专项基金，5 年内建 3 个国家级产业园」，均附《深圳市人民政府关于促进人工智能产业高质量发展的若干措施》原文链接 → **10 分**
- **案例 B**：「GPT-5 Turbo 在 MMLU 达到 95 分」（实际 91.3 分）→ **3 分**（差异超 4%）
- **案例 C**：「据知情人士透露，某大厂 CEO 即将离职」→ **2 分**（无官方源 + 可能涉谣言）

---

## 2. 信源权威度（sourceAuthority）· 权重 10%

信源分级见 [docs/skills/media-industry-standards.md §8](../../../docs/skills/media-industry-standards.md)。

| 分数 | 表现 |
|------|------|
| 9-10 | 主要引用 S 级（新华社 / 人民日报 / 政府官网）≥ 2 条 |
| 7-8 | S + A 级混合 |
| 5-6 | 以 A-B 级为主；C 级适当引用并注明 |
| 3-4 | 以 C 级自媒体为主，无官方源 |
| 0-2 | 只有 D 级（匿名 / 待验证）或无信源 |

### 场景豁免

- `livelihood_zhongcao` / `livelihood_tandian`：允许以个人体验为主，不强求 S 级信源（下调 30% 权重）
- `politics_*`：强制要求 S 级信源 ≥ 2 条，否则自动扣至 3 分以下

---

## 3. 逻辑通顺度（logicFlow）· 权重 10%

| 分数 | 表现 |
|------|------|
| 9-10 | 段落衔接自然，因果链严密，论证层层递进 |
| 7-8 | 整体流畅，1-2 处过渡生硬 |
| 5-6 | 有明显跳跃，需读者脑补 |
| 3-4 | 多处逻辑断裂，需重组 |
| 0-2 | 段落毫无关联，AI 拼凑痕迹 |

### 典型扣分点

- 「与此同时」「另一方面」滥用（句式单一）
- 前后段主题跳跃无过渡（如从政策突然跳到市场）
- 结论与前文论据矛盾

---

## 4. 语言流畅度（languageFluency）· 权重 10%

| 分数 | 表现 |
|------|------|
| 9-10 | 句式多变（长短句交替），标点规范，无赘句 |
| 7-8 | 基本流畅，2-3 处冗余 |
| 5-6 | 机器感较重，5+ 处冗余 |
| 3-4 | 明显翻译腔 / AI 腔 |
| 0-2 | 句式错乱 / 大量病句 |

### AI 腔典型表现（必扣分）

- 过多使用「值得一提的是」「引人深思的是」「在某种程度上」
- 句末滥用「等等」「以及」「此外」
- 过度概括（「各行各业」「方方面面」）
- 每段开头都用「另外」「与此同时」「进一步」

---

## 5. 原创深度（originality）· 权重 10%

| 分数 | 表现 |
|------|------|
| 9-10 | 独立观点 ≥ 1 处、独家数据 ≥ 1 处、视角新颖 |
| 7-8 | 大部分原创，少量借鉴主流观点 |
| 5-6 | 结构和观点与主流媒体趋同 |
| 3-4 | 近乎搬运，新增信息 < 20% |
| 0-2 | 全文复制粘贴多源内容 |

### 检测方法

- 关键观点查 BaiDu / Google 最近 7 天命中率
- 数据点是否有一级信源
- 结论是否包含独立思考（非简单复述）

---

## 6. 结构完整性（structure）· 权重 10%

必须符合对应 `scenario` 的结构要求（见 content_generate §9）。

| 分数 | 表现 |
|------|------|
| 9-10 | 100% 符合子模板结构 |
| 7-8 | 基本符合，1 处偏离 |
| 5-6 | 多处偏离，但核心结构在 |
| 3-4 | 结构混乱，子模板风格不明显 |
| 0-2 | 完全脱离子模板要求 |

### 各子模板核心结构检查项

- `news_standard`：导语（5W1H 覆盖）+ 主体（倒金字塔）+ 背景 + 观察
- `politics_shenzhen`：政策原文 + 要点梳理 + 深度解读 + 意义影响
- `sports_chuanchao`：比分开头 + 上/下半场 + 关键时刻 + 数据表
- `daily_brief`：开篇一句话 + 3-5 条热点 + 结语

---

## 7. 受众价值（audienceValue）· 权重 10%

| 分数 | 表现 |
|------|------|
| 9-10 | 信息密度高、时效性强、对目标受众有实操指导 |
| 7-8 | 有价值但亮点不突出 |
| 5-6 | 信息量偏低，对受众帮助一般 |
| 3-4 | 信息稀薄，可被任意搜索引擎替代 |
| 0-2 | 无价值增量（如纯鸡汤 / 八股） |

---

## 8. 合规性（compliance）· 权重 20%

这是**最高权重**维度。任何硬违规命中 → compliance 0 分 → 综合分一票否决。

| 分数 | 表现 |
|------|------|
| 10 | 0 命中（软 + 硬） |
| 8-9 | 软违规 1 处（如重复使用同一情绪词） |
| 5-7 | 软违规 2-3 处 |
| 1-4 | 硬违规 1 处（广告法极限词 / 模糊政治表述） |
| 0 | 硬违规 ≥ 2 处 / 涉政治红线 / 涉未成年人 |

### 硬违规一票否决清单

- 领导人姓名职务错误
- 广告法极限词（最 / 第一 / 顶级等）
- 未成年人真实身份信息
- 网信办负面词（第 7 条）
- 侮辱诽谤 / 诽谤 / 诽谤性定性（黑哨 / 塌房 / 翻车）

---

## 附录：强制人工复核触发条件

即使综合分 ≥ 95（S 级），以下情况仍建议人工复核：

1. `reviewTier=strict` 且稿件涉及政治 / 未成年人 / 军事
2. 有新奇的结论或独家爆料
3. 引用了 C/D 级信源
4. 单一稿件关联多个热门关键词（有炒作嫌疑）
5. 发布渠道为 app_home 头条 / 首页推荐位

人工复核在 VibeTide 中走 `/admin/review` 队列，编辑手动点击「通过 / 驳回」。
