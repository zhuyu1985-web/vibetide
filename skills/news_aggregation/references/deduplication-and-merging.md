# 新闻聚合 · 去重、事件归档与跨源合并规则

> `news_aggregation` skill 从多个信源（微博 / 头条 / 主流媒体 / 官方门户）
> 聚合新闻条目时，必须去重 + 同事件合并 + 权威信源优先。本文档给出详细规则。

---

## 1. 三层去重策略

### 层 1：URL 完全相同

最快速的去重：两条新闻 URL 归一化后完全相等 → 去重。

归一化规则：
- 去掉 tracking 参数（utm_*, from, spm）
- 去掉 hash（#xxx）
- 协议统一 https
- 尾部斜杠统一去除

### 层 2：标题相似度

```python
def title_similar(t1, t2) -> bool:
    # 字符级 Jaccard
    s1, s2 = set(t1), set(t2)
    jaccard = len(s1 & s2) / len(s1 | s2)
    # 或者 embedding 余弦相似度
    return jaccard > 0.85
```

阈值建议：0.85（保守），高于此认为是同一事件。

### 层 3：正文首段相似度

对于标题党 / 改写的稿件，标题可能不同但正文首段相似：
- 取两篇首段前 200 字
- 计算编辑距离或 BERT embedding 相似度
- 相似度 > 0.9 判定为同事件不同改写版

## 2. 事件归档（Event Clustering）

将多条报道归为同一「事件」，每个事件有唯一 `event_id`：

### 2.1 事件判定 4 要素

两条新闻是否同一事件的判断：

1. **谁**（who）：涉及的主要人物 / 机构是否相同
2. **什么**（what）：事件的核心动作 / 结果是否相同
3. **何时**（when）：时间窗口 ≤ 48 小时
4. **何地**（where）：地点匹配或可以合并

全部匹配 → 同事件；3/4 匹配 → 可能同事件（送人审）。

### 2.2 事件合并效果

```json

  "title_consensus": "深圳发布 AI 产业新政",

  "primary_source": "新华社",
  "report_count": 15,
  "sources": [
    { "platform": "新华社", "tier": "S", "url": "..." },
    { "platform": "人民网", "tier": "S", "url": "..." },

```

## 3. 权威信源优先级

同事件多报道时，保留的「主报道」选择顺序：

1. **最早的官方通稿**（新华社 / 人民日报 / 政府官网）
2. **最早的地方党报党媒**
3. **最权威的垂类媒体**（财经 → 21 财经；科技 → 36Kr；体育 → 新华社体育）
4. **最早的其他 B 级媒体**
5. **最早的 C 级自媒体**

其他报道作为「perspectives」辅助参考。

---

## 4. 跨语言新闻合并（涉外）

来自中文、英文、日文等不同语言的同事件报道：

- 以中文报道为主，外文报道作为「国际视角」标注
- 外文标题 / 摘要必须有中文翻译
- 翻译质量必须二次核验（AI 翻译 + 人工抽查）

### 翻译质量阈值

- `translation_confidence >= 0.9`：直接使用
- `0.7-0.9`：标注「机器翻译」
- `< 0.7`：不合并，保留原文 + 提示翻译失败

## 5. 时效性标签

每条新闻自动标注时效性：

| 标签 | 判定 | 业务意义 |
|------|------|---------|
| 🔥 突发（breaking） | 发布时间 < 1 小时 | 可优先推送 |
| ⚡ 新鲜（fresh） | 1-6 小时 | 常规发布窗口 |
| 📰 今日（today） | 6-24 小时 | 日内仍可跟进 |
| 📖 本周（weekly） | 1-7 天 | 周报可采 |
| 📚 历史（archived） | > 7 天 | 不作为新闻采用 |

## 6. 分类标签

标签体系（参照新华社分类 + 微博 v2 分类）：

### 一级分类
- 时政 / 国际 / 军事
- 经济 / 产业 / 金融
- 科技 / 互联网 / AI
- 社会 / 民生 / 地方
- 文化 / 教育 / 体育
- 娱乐 / 明星 / 影视
- 健康 / 医疗 / 养生

### 二级分类示例

```
科技
  ├── AI / 大模型
  ├── 硬件 / 芯片
  ├── 互联网 / 平台
  ├── 5G / 通信
  └── 新能源 / 电动汽车

经济
  ├── 宏观 / 政策
  ├── 股市 / 基金
  ├── 房地产
  ├── 消费 / 零售
  └── 企业 / 财报
```

### 自动分类方法

- **规则法**：基于关键词清单 + 正则
- **模型法**：中文 BERT / roberta 微调
- **混合**：先规则命中，未命中走模型

## 7. 黑名单过滤

自动剔除的新闻类型：

- **广告 / 营销**：标题含「推荐」「首发」「抢购」；或内容付费推广标识
- **谣言 / 不实信息**：经权威机构辟谣的话题
- **低质量自媒体**：账号认证等级低 + 转发率 > 90%
- **重复营销号**：同一文本在多个账号扎堆发布
- **境外反华内容**：命中政治敏感词清单

## 8. 地域归档

按新闻地域归档（便于 `local_news` 工作流消费）：

```json
{
  "geo_scope": "city",
  "country": "中国",
  "province": "广东",
  "city": "深圳",
  "district": "南山区"
}
```

判定方式：
- 标题 / 正文包含地名 → 精确匹配
- 政府官网发布 → 取对应行政区划
- 多地并列 → scope=region 或 national

## 10. 与下游的衔接

- `topic_extraction`：从聚合结果中提取可创作的选题
- `heat_scoring`：为每个事件计算热度评分
- `content_generate`：以 event 为单位喂入 prompt
- `fact_check`：跨源交叉验证的原始数据支持
