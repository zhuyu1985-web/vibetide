## 使用规范

### 交叉验证组合

针对不同领域事实，推荐的 3 源组合：

- **时政**：新华社 + 人民日报 + 国务院官网
- **经济**：国家统计局 + 人民银行 + 第一财经（B 级佐证）
- **科技**：工信部 + 中国信通院 + 36Kr（B 级佐证）
- **汽车**：中汽协 + 乘联会 + 21 世纪经济报道
- **体育（川超）**：川超官网 + 新华社体育 + 四川日报
- **国际**：新华社国际 + BBC + Reuters

### 不可信源（D 级）

以下源在任何场景都**不得**作为事实来源：

- 匿名博客 / Reddit / 4chan
- 微博未认证账号
- 抖音 / 快手未认证账号
- 「知情人士」「业内爆料」（无机构署名）
- 维基百科（可作为背景参考，不可作为事实源）
- Twitter / X 除非政府 / 媒体认证账号

### 特殊场景信源

- **涉军事**：仅中央军委 + 国防部 + 新华社军事
- **涉港澳台**：仅中央对港澳办 + 国台办 + 新华社
- **涉未成年人**：仅教育部 + 民政部 + 中央媒体
- **涉股市**：仅证监会 + 上交所 + 深交所 + 财新 / 第一财经

## API 端点（VibeTide 集成）

当前 fact_check 自动接入的 API：

- **Tavily Search**（`TAVILY_API_KEY`）：全网搜索，返回 S/A 级源优先
- **Jina Reader**（`JINA_API_KEY`）：网页深读，可解析政府门户原文
- **KB retrieval**：命中本地「官方信源白名单」KB，无需走 web

每次 web 调用都记录到 `mission_artifacts`，便于审计和反向追溯。
