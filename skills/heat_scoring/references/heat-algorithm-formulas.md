# 热度评分 · 算法公式与行业基准

> `heat_scoring` skill 用于为话题 / 事件计算综合热度分（0-100）。本文档
> 提供具体公式、维度权重、行业基准值、异常处理规则。

---

## 1. 核心评分公式

综合热度分采用 4 维加权：

```
HeatScore = 0.35 × SearchVolume + 0.25 × Engagement + 0.25 × MediaCoverage + 0.15 × Timeliness

其中（归一化到 0-100）：
- SearchVolume  = log10(1 + 百度/微博搜索指数) × scale  
- Engagement    = log10(1 + 互动总量) × scale  
- MediaCoverage = 报道平台数 × 10 + 平台权重和 × 5  
- Timeliness    = max(0, 100 − elapsed_hours × 2)  
```

---

## 2. 4 维度详解

### 2.1 搜索量（SearchVolume）· 权重 35%

来源：
- 百度搜索指数
- 微博搜索次数
- 抖音搜索次数
- Google Trends（涉外话题）

取 Top 3 平台的搜索量最大值后 log10 变换 → 再线性映射到 0-100。

**基准值**（参考 2025 年数据）：
- 破圈级（百度指数 > 100 万）：SearchVolume = 100
- 现象级（10-100 万）：80-95
- 热议级（1-10 万）：60-80
- 一般级（1 千-1 万）：40-60
- 小众级（< 1 千）：< 40

### 2.2 互动量（Engagement）· 权重 25%

各平台互动统一归一化：

- 微博：(转发 × 3) + (评论 × 2) + (点赞 × 1)
- 抖音：(评论 × 3) + (分享 × 2) + (点赞 × 0.1)
- 知乎：(回答 × 5) + (评论 × 2) + (点赞 × 1)
- 微信公众号：(在看 × 5) + (评论 × 3) + (阅读 × 0.01)

**基准值**：
- 百万级互动：100
- 10 万级：80
- 万级：60
- 千级：40
- < 千：< 30

### 2.3 媒体覆盖（MediaCoverage）· 权重 25%

报道平台数 + 权威加权：

```python
def media_coverage_score(event):
    platform_count = len(event.sources)
    tier_weights = { "S": 10, "A": 7, "B": 4, "C": 2, "D": 0.5 }
    weight_sum = sum(tier_weights[s.tier] for s in event.sources)
    return min(100, platform_count * 5 + weight_sum)
```

**基准值**：
- 全网覆盖（10+ S 级）：100
- 主流覆盖（5-10 S/A 级）：80-95
- 较广覆盖（3-5 S/A/B）：60-80
- 有限覆盖（1-3 B/C）：40-60
- 孤证（仅 1 个 C/D 源）：< 30

### 2.4 时效性（Timeliness）· 权重 15%

基于事件首次发生时间：
- 1 小时内：100
- 1-3 小时：90
- 3-6 小时：80
- 6-12 小时：70
- 12-24 小时：50
- 24-48 小时：30
- > 48 小时：< 20

## 3. 加分项 / 扣分项

### 3.1 加分项

| 条件 | 加分 |
|------|------|
| 官方权威源首发（新华社 / 政府官网） | +10 |
| 跨领域扩散（如科技事件进入时政讨论） | +8 |
| 高质量长文解读数 ≥ 5 篇 | +5 |
| 事件延续性强（连续 3 天 Top 10） | +5 |

### 3.2 扣分项

| 条件 | 扣分 |
|------|------|
| 热度异常（Z-score > 5，疑似刷单） | -20 |
| 单平台孤立（只在 1 个小众平台发酵） | -10 |
| 含负面词 / 谣言标记 | -5 ~ -20（视严重程度） |
| 已被权威辟谣 | -100（清零） |

## 4. 行业基准值（2026 年 4 月）

### 4.1 突破分数线

- **HeatScore ≥ 95**：现象级事件（如 GPT-5 发布 / 苹果新品）
- **85-94**：全网热议（如某地政策重大变动）
- **75-84**：行业内热话题
- **60-74**：垂类关注
- **45-59**：一般资讯
- **< 45**：不建议深度跟进

## 6. 趋势评估

除分数外，附加趋势预测：

```python
def trend_forecast(score_history: list[float]):
    if score_history[-1] > score_history[-3] * 1.2:
        return "📈 上升"
    elif score_history[-1] < score_history[-3] * 0.8:
        return "📉 下降"
    elif all(abs(score_history[-1] - s) < 5 for s in score_history[-3:-1]):
        return "➡️ 稳定"
    else:
        return "〰️ 波动"
```

## 7. 异常处理

### 7.1 数据源降级

某个数据源不可用：
- 对应维度的 confidence 降低
- 权重自动重新分配到其他维度
- 输出 `reliability: "partial"` 提示用户

### 7.2 极端值异常

- HeatScore 为 0：检查数据是否全部缺失
- HeatScore > 100：clip 到 100；调查公式是否溢出
- 连续 10 次事件都是 S 级：检查是否基准值过低（需要重校准）

---

## 8. 定期校准

基准值应每季度校准一次：

- 抽样 1000 个历史事件
- 重新计算分布（分位数 / 平均值）
- 调整等级门槛（确保 S 级占 1%，A 级占 9%，B 级占 30%...）
