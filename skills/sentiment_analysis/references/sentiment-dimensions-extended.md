# 情感分析 · 维度细分与政治中立判定

> `sentiment_analysis` skill 需要在中文语境下做多维情感分析，特别要处理
> 政治敏感 / 国际关系 / 民族宗教话题的中立性判定。本文档给出详细判定规则。

---

## 1. 5 层情感维度

传统二分（正/负）不足以刻画中文内容情感。本 skill 用 5 层：

| 层 | 维度 | 值域 |
|---|------|------|
| L1 | 基础极性 | positive / negative / neutral / mixed |
| L2 | 情绪强度 | 0.0-1.0（0 = 完全平淡，1 = 极端情绪化） |
| L3 | 情绪类型 | joy / anger / sadness / fear / surprise / disgust / trust / anticipation |
| L4 | 表达方式 | 理性陈述 / 讽刺反语 / 情绪宣泄 / 批评建议 |
| L5 | 政治/商业立场 | 中立 / 偏向性 / 立场鲜明 |

---

## 2. L1 基础极性判定规则

### positive（正面）
- 含赞美 / 肯定 / 期待关键词
- 主语情绪指向「好的方向」
- 典型：「取得突破」「令人振奋」「前景看好」

### negative（负面）
- 含批评 / 担忧 / 不满关键词
- 主语情绪指向「坏的方向」
- 典型：「引发担忧」「饱受诟病」「形势严峻」

### neutral（中立）
- 纯客观陈述事实
- 无情感指向词
- 典型：「发布了新政策」「数据显示 X%」

### mixed（复杂）
- 同时包含正负两方面
- 典型：「政策利好科技股但加剧能源行业压力」

## 3. 政治话题中立性判定（核心）

中文政治话题的情感判定需要额外谨慎。误判可能导致合规风险。

### 3.1 必须判 neutral 的情况

- 党和国家领导人的官方表态 / 讲话 / 重要批示
- 政策文件原文引用
- 国务院 / 部委正式发布的白皮书
- 新华社 / 人民日报的官方通稿

**这些内容不论字面情感如何，AI 必须判 `neutral` + `rational statement`**。

### 3.2 不得判「负面」的情况

- 对国家大政方针的客观分析
- 对历史事件的正面纪念
- 对重大成就的赞美（如脱贫攻坚、抗疫、航天）
- 涉及民族团结 / 宗教和谐的表述

**AI 对这些话题倾向「负面」的，可能是模型偏见，必须校正为 neutral 或 positive。**

### 3.3 敏感话题的特别处理

- 涉港澳台：严格按《中央政府涉港澳台报道用语规范》判定
- 涉外关系：中立陈述，不参与「好 / 坏」判定
- 民族宗教：一律 neutral
- 涉及重大事故 / 灾害：neutral，不放大情绪

## 4. 商业舆情立场判定

### 4.1 企业正面 / 负面

企业话题情感三级：

- **正面**：业绩增长 / 获奖 / 产品好评 / 上市成功
- **负面**：业绩下滑 / 危机 / 产品召回 / 股价大跌
- **中立**：产品发布（无评价）/ 行业陈述

### 4.2 产品评价

- 用户体验类：直接按用户情绪分类
- 比较类（「A vs B」）：取决于比较倾向
- 测评类：按测评者立场（KOL / 独立媒体 / 厂家软文分别判）

### 4.3 股价敏感

企业相关情感可能影响股价，判定要**留痕**：
- 所有 negative 判定必须有具体依据（引用原文）
- 不得基于猜测 / 传言判负面
- 争议性事件 → mixed 而不是 negative

---

## 5. 舆情细分（8 种情绪）

基于 Plutchik 情绪轮的 8 基本情绪 + 中文本土化：

| 情绪 | 典型表达 | 业务场景 |
|------|---------|---------|
| 喜悦（joy） | 欣喜 / 兴奋 / 自豪 | 成就报道 / 喜讯 |
| 愤怒（anger） | 气愤 / 声讨 / 抗议 | 维权 / 丑闻 |
| 悲伤（sadness） | 惋惜 / 痛心 / 哀悼 | 事故 / 逝去 |
| 恐惧（fear） | 担忧 / 紧张 / 焦虑 | 疫情 / 危机 |
| 惊讶（surprise） | 震惊 / 出乎意料 | 黑天鹅事件 |
| 厌恶（disgust） | 鄙视 / 嫌弃 / 反感 | 劣迹 / 丑闻 |
| 信任（trust） | 信赖 / 认同 / 支持 | 品牌忠诚度 |
| 期待（anticipation） | 憧憬 / 期盼 / 预测 | 新品预热 / 展望 |

## 6. 讽刺反语识别

中文网络语境下讽刺反语特别常见。需要识别：

### 6.1 典型讽刺信号

- 正面词 + 上下文反转（「这服务真棒，等了 3 小时」）
- 反讽感叹号（「绝了！」「真的吗？？？」）
- 网络梗（「真诚不伪造」= 伪造）
- 引号加重（「『贴心』服务」）

### 6.2 处理策略

- 识别出讽刺 → L1 判 negative，L4 标注「讽刺反语」
- 不得机械取表层情感词
- 如不确定 → 降低置信度 + 标注「可能讽刺」

---

## 7. 多观点共存判定（mixed）

一篇稿件里可能同时包含正负观点：

```
「特斯拉 Q1 销量同比增长 20%（正面），但市场预期为 25%，实际低于预期（负面）」
```

判定：
- L1 = `mixed`
- `aspects`:
  - `sales_growth`: positive（销量增长）
  - `expectation_gap`: negative（低于预期）
- L2 强度取较高者

---

## 8. 细粒度方面情感（ABSA）

针对一个话题的多个方面分别打分：

```json
{
  "topic": "iPhone 17 发布",
  "overall": "positive",
  "aspects": {
    "camera": { "sentiment": "positive", "score": 0.85, "quote": "摄像能力大幅提升" },
    "battery": { "sentiment": "neutral", "score": 0.5, "quote": "续航基本同比" },
    "price": { "sentiment": "negative", "score": 0.3, "quote": "定价偏高" }
  }
}
```

---

## 9. 置信度阈值

| 置信度 | 处理 |
|-------|------|
| ≥ 0.85 | 直接采用 |
| 0.7-0.85 | 标注「高置信」但仍交付 |
| 0.5-0.7 | 标注「中等置信」 + 必要时人工复审 |
| < 0.5 | 不可靠；输出「无法判定」 |

## 10. 边界场景

### 9.1 数据类内容（纯数字）

「2025 年 GDP 增长 5.2%」

- 纯数字本身是 neutral
- 但上下文「高于预期 4.8%」 → positive
- 上下文「低于去年 5.5%」 → negative

### 9.2 引用类内容

「专家 A 认为 X 是好事（positive），但专家 B 认为 X 有风险（negative）」

- overall: `mixed`
- 分别标注 quoted_entity

### 9.3 新闻标题

标题情感通常更极端，需要：
- 单独分析（不影响正文情感）
- 与正文对比：标题更负面 → 疑似标题党

## 12. 禁止的判定

- **不得**把官方表态 / 政策原文判为 negative
- **不得**基于单个情绪词快速判定（必须看上下文）
- **不得**把中性陈述误判为 positive / negative
- **不得**对涉外 / 港澳台话题做倾向性判定
