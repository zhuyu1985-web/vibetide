# Proposal: Refine Skill Management UX

## Change ID
`refine-skill-management-ux`

## Status
`proposed`

## Summary

参考 SkillsMP.com 的技能市场设计模式，优化技能管理的列表页和详情页体验。将技能详情从弹窗升级为独立页面（`/skills/[id]`），以便完整展示 Markdown 文档；列表页改为更紧凑的行卡片布局，增加排序选项。

## Motivation

当前技能管理页面存在以下体验问题：

1. **详情展示受限**：技能的 Markdown 内容只能在弹窗中查看，弹窗宽度和高度有限，无法舒适阅读长文档
2. **列表信息密度低**：3 列网格卡片布局中每张卡片信息有限，描述只能显示 2 行
3. **缺少排序**：无法按绑定数、更新时间、名称排序
4. **无独立 URL**：查看技能详情没有独立 URL，无法分享链接

参考 SkillsMP.com 的设计：
- 列表页使用**行卡片布局**（每行一个技能），信息更完整
- 每个技能有**独立详情页**，完整渲染 Markdown 文档
- 提供 **Stars / Recent 排序切换**
- 搜索框突出展示

## Scope

### In Scope
- 技能列表改为行卡片布局（每行一条，左侧名称+元信息，右侧描述，类似 SkillsMP 风格）
- 新增排序选项：按绑定数、按更新时间、按名称
- 点击技能卡片跳转到独立详情页 `/skills/[id]`
- 详情页：左侧主区域渲染完整 Markdown，右侧 sidebar 显示元数据（分类、版本、类型、绑定员工数、创建/更新时间）
- 详情页提供编辑和删除操作按钮
- 列表页移除查看详情弹窗（用页面跳转替代）

### Out of Scope
- 技能收藏/点赞功能（SkillsMP 有 stars 和 favorites，本期不做）
- 技能来源/作者概念（我们是单租户技能库，不需要）
- 搜索的 AI 语义匹配（保持关键词搜索）
