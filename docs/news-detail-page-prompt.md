# Vibe Coding Prompt：新闻稿件详情阅读页 & 富文本编辑器

> 技术栈：Next.js 14+ (App Router) + TypeScript + Tailwind CSS + Tiptap (富文本编辑器) + Supabase
> 本文档用于指导大模型进行 vibe coding 开发，请严格按照以下功能描述实现。

---

## 一、页面定位与核心理念

这是产品最核心的生产力界面——**新闻稿件详情页**，集沉浸阅读、富文本编辑、AI 辅助分析、深度笔记于一体的"第二大脑"工作台。

### 核心设计原则

1. **双模切换**：页面默认进入 **阅读模式 (Read Mode)**，用户点击"编辑"按钮后切换为 **编辑模式 (Edit Mode)**，正文区域变为富文本编辑器。
2. **多态路由 (Polymorphic Routing)**：根据内容元数据 `type` 字段自动判断，加载不同的渲染器（图文模式 vs 视频模式）。
3. **AI 伴读伴写**：左侧智能对话面板贯穿阅读与编辑全流程，AI 既能解读内容，也能直接操作编辑器（插入、替换、润色）。
4. **记者工学**：强调引用规范、证据留存（快照/存档）、双向联动、多任务对比（悬浮批注）。

## 二、页面布局架构

采用响应式 **三栏布局 (3-Column Layout)**：

```
┌──────────────────────────────────────────────────────┐
│                    顶部导航栏 (Header)                 │
├────────────┬──────────────────────┬───────────────────┤
│   左侧栏    │     中间核心区域       │     右侧栏        │
│  (20-25%)  │     (45-55%)         │    (25-30%)       │
│            │                      │                   │
│  智能对话   │  阅读模式：纯净正文    │  AI 解读 / 批注    │
│  AI Chat   │  编辑模式：富文本编辑器  │  / 听记(视频)     │
│  Panel     │                      │                   │
├────────────┴──────────────────────┴───────────────────┤
│                    底部状态栏（可选）                    │
└──────────────────────────────────────────────────────┘
```

- **全屏禅模式 (Zen Mode)**：一键收起左右侧栏，仅保留中间核心区域。
- **面板折叠**：左右侧栏均可独立折叠/展开。
- **阅读进度条**：顶部边缘极细进度条，实时展示阅读/播放进度。

## 三、内容类型路由 (Content Router)

页面加载时读取笔记元数据，决定渲染模式：

### 模式 A：图文阅读模式 (Text Mode)
- **触发条件**：`type = article` 或 `type = text/html`
- **特征**：强调排版、文字高亮、文档大纲、富文本编辑

### 模式 B：视听分析模式 (Video/Audio Mode)
- **触发条件**：`type = video`（来源为抖音、B站、YouTube 等或包含主视频流）
- **特征**：强调播放控制、ASR 听记、时间戳锚点

### 混合内容处理
- 图文中嵌入视频：默认走图文模式，视频旁提供"提取为视频笔记"按钮
- 点击后将视频单独提取，可切换到视频模式深度分析

## 四、顶部导航栏 (Header) — 全局通用

### 4.1 左侧：导航与路径
- **返回按钮**：返回列表页
- **面包屑导航**：展示分类路径层级

### 4.2 中间：视图切换 (View Switcher)
采用 Segment Control（分段控制器）样式，支持快捷键切换：

| 视图 | 说明 |
|------|------|
| **沉浸阅读 / 播放器** | 默认。清洗后的纯净正文或智能播放器 |
| **原始网页 (Web View)** | iframe 加载原始 URL，保留原站排版 |
| **AI 速览 (AI Brief)** | AI 生成的结构化事实卡片（核心事实、人物、时间、地点、结果），极简排版 |
| **网页存档 (Web Archive)** | 永久保存的网页快照（HTML/图片），用于证据留存 |

### 4.3 右侧：工具箱 (Toolbar)

所有按钮以 Icon 形式展示，Hover 显示文字提示。

#### A. 阅读/编辑模式切换按钮
- **阅读模式 (Read)**：默认态，正文只读渲染
- **编辑模式 (Edit)**：点击后正文区域切换为 Tiptap 富文本编辑器
- 切换时保持滚动位置不变
- 编辑模式下显示保存/取消按钮

#### B. 阅读器样式 (Appearance)
- **图文模式**：字号 (`A-`/`A+` 5-7档)、页边距（窄/标准/宽）、行高（紧凑/舒适/宽松）、主题色（亮/暗/护眼/跟随系统）、字体（系统默认/衬线/无衬线/等宽）
- **视频模式**：字幕大小、字幕背景透明度、播放器背景填充模式（黑边 vs 高斯模糊）

#### C. 更多操作 (Actions Menu)
下拉菜单包含：

**分享与访问：**
- 分享阅读：生成分享卡片或公开链接
- 访问原始网页：跳转浏览器打开
- 更新/保存快照：手动触发网页存档

**复制功能：**
- 复制原链接
- 复制 Markdown 链接
- 复制纯文本 (Plain Text)
- 复制 Markdown（保留格式）
- 复制 HTML
- 复制快照 HTML（带样式卡片代码）
- **复制为引用 (Copy as Citation)**：格式 `> 选中文本 [来源](URL) (访问于 日期)`
- 复制 ASR 逐字稿（视频模式特有）

**导出功能：**
- 导出 PDF（打印样式优化）
- 导出 Markdown
- 导出 TXT
- 导出快照图片/HTML
- 导出 SRT 字幕（视频模式特有）
- 导出视频关键帧打包（视频模式特有）
- 导出网页存档 HTML

**整理功能：**
- 设为星标 (Toggle Star)
- 移动文件夹 (Move Category)
- 编辑元信息 (Edit Meta)：弹窗修改标题、摘要、标签、原始发布时间
- 归档 (Archive) / 稍后读
- 删除 (Delete)

#### D. AI 解读按钮
- 点击后右侧面板自动展开并选中"AI 解读"Tab
- 若尚未生成，触发流式生成；若已生成，直接展示
- 图标建议：魔法棒或芯片

#### E. 批注按钮
- 点击后右侧面板展开并选中"批注列表"Tab
- 按钮右上角数字角标显示当前文章批注总数
- 图标建议：高亮笔

## 五、左侧区域：智能对话面板 (AI Chat Panel)

> **重要变更**：左侧栏从原设计的"智能大纲导航"改为"智能对话面板"，作为全页面的 AI 交互中枢。

### 5.1 面板结构

```
┌─────────────────────┐
│  ✦ AI 助手    [折叠] │  ← 面板标题栏
├─────────────────────┤
│  [大纲] [对话] [历史] │  ← Tab 切换
├─────────────────────┤
│                     │
│   对话消息列表        │  ← 主内容区（可滚动）
│   - AI 消息          │
│   - 用户消息          │
│   - 操作卡片          │
│                     │
├─────────────────────┤
│  快捷指令栏           │  ← 常用 AI 指令快捷按钮
├─────────────────────┤
│  [📎] 输入框... [发送]│  ← 底部输入区
└─────────────────────┘
```

### 5.2 Tab 功能

#### Tab 1：智能大纲 (Outline)
- 自动提取文章 H1/H2/H3 标签生成目录树
- 若正文无标题标签，AI 自动提炼关键段落的小标题作为锚点
- 点击锚点，正文平滑滚动至对应位置
- 滚动正文时，对应目录项实时高亮（阅读进度联动）
- 极短文章自动隐藏大纲，显示提示"文章较短，无需导航"
- **视频模式下**：变为"视频章节 (Smart Chapters)"
  - AI 基于语义/画面转场自动分段（如 `00:00 导语` → `01:30 现场采访` → `03:45 专家分析`）
  - 点击章节标题 → 视频跳转 + 右侧逐字稿同步跳转

#### Tab 2：AI 对话 (Chat) — 核心功能
这是整个面板的核心，支持基于当前文章/编辑内容的上下文对话。

**对话能力：**
- 基于当前文章全文的 RAG 问答（"这篇文章对房地产有何利空？"）
- 对选中文本的解释、翻译、扩写、缩写
- 事实核查与背景补充
- 多轮对话，支持上下文连续追问

**与编辑器的交互能力（编辑模式下）：**

当处于编辑模式时，AI 对话具备以下编辑器操作能力：

| 指令类型 | 示例 | 编辑器行为 |
|---------|------|-----------|
| **插入内容** | "帮我在第三段后插入一段过渡语" | AI 生成内容，用户确认后插入到编辑器指定位置 |
| **替换选中** | 选中一段文字后说"帮我润色这段" | AI 生成替换内容，以 diff 形式展示，用户确认后替换 |
| **全文润色** | "帮我润色全文，使其更加专业" | AI 逐段生成润色建议，用户可逐条接受/拒绝 |
| **续写** | "帮我续写下一段" | AI 基于上下文生成续写内容，光标位置插入 |
| **改写风格** | "把这篇改成新华社通稿风格" | AI 生成全文改写，以 diff 对比展示 |
| **生成摘要** | "生成一段 100 字的摘要" | AI 生成后可一键填入元信息的摘要字段 |
| **提取要素** | "提取文中所有人名和机构名" | AI 生成结构化列表，可一键插入为标签 |
| **翻译** | "翻译成英文" | AI 翻译全文或选中段落 |

**AI 操作卡片 (Action Card) 设计：**

当 AI 生成了可操作内容时，在对话流中展示为特殊卡片：

```
┌─────────────────────────┐
│ ✦ AI 生成内容             │
│                         │
│ 「润色后的段落文字...」    │
│                         │
│ [应用到编辑器] [复制] [重新生成] │
└─────────────────────────┘
```

- **应用到编辑器**：将内容插入/替换到编辑器中（带 undo 支持）
- **Diff 预览**：对于替换操作，展示红绿 diff 对比
- **批量操作**：全文润色时支持"全部接受"/"逐条审核"

**快捷指令栏 (Quick Commands)：**

输入框上方显示一排可滚动的快捷指令 Chip：

- 阅读模式快捷指令：`总结全文` `提取金句` `分析立场` `生成时间线` `翻译全文` `事实核查`
- 编辑模式快捷指令：`润色选中段落` `续写下文` `生成标题` `缩写为摘要` `扩写详述` `改为正式语体`
- 视频模式快捷指令：`总结视频` `提取关键帧描述` `生成文字稿` `识别说话人`

快捷指令根据当前模式（阅读/编辑/视频）动态切换显示。

#### Tab 3：对话历史 (History)
- 展示当前文章的所有历史 AI 对话记录
- 按时间分组，支持搜索
- 点击历史记录可恢复上下文继续对话

### 5.3 选中文本与对话联动

- **阅读模式**：用户在正文选中文字后，选中内容自动填入对话输入框作为引用上下文，用户可直接追问
- **编辑模式**：用户在编辑器中选中文字后，左侧面板自动切换到对话 Tab，选中内容作为上下文，并展示编辑相关的快捷指令（润色/改写/翻译等）

### 5.4 面板折叠与响应式

- 支持点击折叠按钮收起左侧面板
- 折叠后在左边缘显示一个悬浮的 AI 图标按钮，点击展开
- 移动端：左侧面板变为底部抽屉 (Bottom Sheet) 或浮层

## 六、中间区域：自适应核心舞台 (Main Stage)

### 6.1 场景 A：图文模式 (Text Mode)

#### 阅读模式 (Read Mode) — 默认

**元信息头 (Meta Header)：**
- 标题：大字号展示
- 来源信息：来源媒体图标 + 媒体名称 + 作者名
- 时效性信息：发布时间 + 抓取时间（记者刚需，区分时效）
- 标签区：展示现有标签，支持点击快速增删
- 阅读预估："约 X 分钟读完"

**正文渲染：**
- 纯净排版：去除广告侧边栏，保留图文
- 代码块：语法高亮 + 一键复制按钮
- 图片：保留图注说明文字
- 表格：响应式表格渲染

**正文交互：**

*划词气泡菜单 (Pop-over Menu)* — 高频交互：

用户选中文字后，鼠标附近立即悬浮微型工具条：
- **高亮色块**：红 / 黄 / 绿 / 蓝 / 紫（默认选中上次使用的颜色）
- **写批注 (Pen Icon)**：点击后右侧面板展开并聚焦输入框
- **AI 解释 (Magic Icon)**：左侧面板切换到对话 Tab，自动发起"解释这段话"
- **搜索 (Search Icon)**：搜索选中关键词
- **复制 (Copy Icon)**：仅复制选中文本

*图片查看 (Lightbox)：*
- 点击正文图片全屏查看
- 支持旋转、下载、以图搜图

#### 编辑模式 (Edit Mode)

点击顶部"编辑"按钮后，中间区域切换为 Tiptap 富文本编辑器。

**编辑器功能清单：**

*基础格式：*
- 标题层级：H1 / H2 / H3 / H4
- 文本格式：加粗、斜体、下划线、删除线、上标、下标
- 对齐方式：左对齐、居中、右对齐、两端对齐
- 文字颜色、背景色
- 字号调整

*结构元素：*
- 有序列表、无序列表、任务列表（勾选框）
- 引用块 (Blockquote)
- 分割线 (Horizontal Rule)
- 代码块（支持语言选择和语法高亮）
- 表格（插入、合并单元格、调整列宽）

*富媒体：*
- 图片插入（上传/URL/粘贴）、图片大小调整、图注编辑
- 视频嵌入（URL 解析）
- 文件附件
- 嵌入 iframe

*高级功能：*
- 链接插入与编辑
- 脚注
- 目录自动生成 (TOC)
- 字数/字符统计（底部状态栏）
- Markdown 快捷输入（输入 `#` 自动转为标题、`-` 转为列表等）
- 拖拽排序段落/块级元素
- 撤销/重做 (Undo/Redo)

*协作与版本：*
- 自动保存（防抖 debounce，编辑停止 3 秒后自动保存）
- 手动保存快捷键 `Ctrl+S` / `Cmd+S`
- 编辑历史/版本对比（可选）

*工具栏设计：*
- 编辑器顶部固定浮动工具栏 (Floating Toolbar)
- 支持"/"命令菜单 (Slash Command)：输入 `/` 弹出快捷插入菜单
- 选中文本时显示 Bubble Menu（浮动格式工具栏）

**编辑模式下的 AI 交互增强：**
- 选中文本后，Bubble Menu 中增加 **AI 按钮**，点击后左侧面板自动带入选中文本发起对话
- 支持 `/ai` 命令：在编辑器中输入 `/ai 润色这段` 可直接触发 AI 操作
- AI 生成内容插入编辑器时，短暂高亮标记（如淡紫色背景 2 秒后消退），让用户知道哪些是 AI 写入的

### 6.2 场景 B：视频模式 (Video Mode)

中间区域变为**智能播放器容器**。

**自适应画幅：**
- 横屏新闻 (16:9)：居中，黑色背景填充
- 竖屏新闻 (9:16)：居中，左右高斯模糊填充背景，适应桌面端

**手势/快捷键：**
- `空格` 暂停/播放
- `←/→` 快退/快进 5 秒
- `↑/↓` 音量调节

**生产力工具栏 (进度条下方)：**
- 倍速播放：0.5x / 0.75x / 1.0x / 1.25x / 1.5x / 2.0x / 3.0x
- 一键截帧 (Snapshot Frame)：相机图标，截取当前画面，自动保存到右侧批注列表并附带时间戳
- 画中画 (PiP)：小窗播放，用户可滚动查看其他面板
- 循环播放区间 (Loop Section)：在进度条上框选一段反复循环（用于听写校对）
- CC 字幕开关
- 静音阅读模式：默认静音播放，依赖右侧高亮逐字稿阅读

## 七、右侧区域：智库面板 (Right Sidebar)

面板顶部通过 Tab 切换不同功能模块。

### 7.1 Tab：AI 解读 (AI Analysis)

#### 图文模式下的 AI 解读

**多视角分析（Dropdown 切换）：**

| 视角 | 内容 |
|------|------|
| **摘要模式 (TL;DR)** | 默认。一句话核心摘要 + 3-5 个关键要点 (Key Takeaways) |
| **记者视点** | 分析消息源可靠性、报道偏见、利益相关方 |
| **金句提取** | 提取文中有价值的引述和关键语句 |
| **时间线** | 梳理事件发展脉络，按时间排列 |
| **关键问答** | AI 提炼 3 个核心问题 + 精准回答 + 未回答的问题/逻辑漏洞 |
| **深度剖析** | 利益相关方分析表 (Who/Did What/Impact) + 数据透视表 + 底层逻辑分析 |

**AI 解读内容展示：**
- Markdown 渲染
- 支持流式输出 (Streaming)，生成中显示 Loading 动画
- 情感/立场标签：`客观中立` / `看涨` / `批判性` / `软广嫌疑`
- 关键结论支持点击定位到原文对应段落

**缓存策略：**
- 公共文章的 AI 解读结果存入数据库
- 第二个用户点击时直接读取，秒开且省 Token

#### 视频模式下的 AI 解读（升级版）

在图文模式基础上增加：
- **视觉摘要 (Visual Summary)**：识别画面关键帧（"00:45 出现火灾现场画面"、"02:10 展示数据图表"）
- **金句/关键帧提取**：列出关键语句，点击可跳转播放
- **说话人识别摘要**：按说话人整理发言要点
- **Deepfake 预警**：提示 AI 生成痕迹或剪辑拼接痕迹（辅助核查）

### 7.2 Tab：批注列表 (Annotations) — 核心功能

按在文中出现的位置顺序排列卡片。

**批注卡片 (Note Card) 结构：**

```
┌──────────────────────────────┐
│ █ 「被高亮的原文引用...」      │  ← 左侧带高亮颜色竖线
├──────────────────────────────┤
│ 用户写的批注内容...            │  ← 支持为空（仅高亮）
├──────────────────────────────┤
│ [改色] [复制] [删除] [📌浮顶] │  ← 操作栏
└──────────────────────────────┘
```

**锚点行为：**
- 图文模式：点击卡片 → 正文滚动到对应段落并闪烁高亮
- 视频模式：点击卡片 → 视频跳转到对应秒数 (Timecode)；显示截帧图片或对应台词

**双向联动 (Bi-directional Linking)：**
- 点击右侧批注 → 正文跳转到对应位置
- 点击正文高亮区域 → 右侧面板自动滚动到对应卡片并呈选中态

**复制格式：**
- 复制"引用原文 + 笔记内容"
- 视频批注复制时附带时间戳

**特色功能：浮顶 (Floating Sticky Note)**

- **触发**：点击卡片上的图钉图标
- **效果**：
  1. 卡片从右侧列表脱离
  2. 变为半透明悬浮窗，置于页面最上层 (Z-Index 最高)
  3. 可拖拽到屏幕任意位置（如拖到正文旁对比阅读）
  4. 全局驻留：切换章节/滚动页面时依然可见
- **归位**：点击悬浮窗"取消浮顶"或"关闭"，卡片飞回列表原位
- **价值**：跨文章漫游时依然可见，便于跨多篇报道写稿对比

### 7.3 Tab：听记 (Transcript) — 视频模式特有

展示视频 ASR 转写的逐字稿。

**说话人区分 (Speaker Diarization)：**
- AI 自动识别不同角色：`说话人 1`、`主持人`、`采访对象`
- 支持手动重命名（将 `说话人 1` 改为具体人名），全文自动替换

**音文同步 (Audio-Text Sync) — 核心交互：**
- **点击文字跳转视频**：点击任意一句台词，视频跳转到该时间点
- **播放高亮跟随**：视频播放时，逐字稿呈"卡拉OK式"逐词/逐句高亮滚动

**文本操作：**
- 划词批注：像处理普通新闻一样在逐字稿上划词、高亮、批注（自动记录时间戳）
- 复制文本 / 复制文本+时间戳（格式：`[02:15] 这是一个关键声明`）

**校对模式：**
- 允许用户直接修改 ASR 识别错误的文字
- 修改后导出字幕时应用修正内容

**导出：**
- 导出 SRT（标准字幕文件）
- 导出 TXT（纯文本）
- 导出 Word（包含时间戳和说话人的文档）

### 7.4 底部：AI 追问输入框

所有 Tab 底部共享一个输入框，支持针对当前内容的 AI 追问 (Chat with Article)。
- 与左侧 AI 对话面板共享上下文
- 此处发起的对话也会同步到左侧面板的对话 Tab

```
阅读模式 (默认)
    │
    ├── 点击「编辑」按钮
```

## 九、边缘情况与特殊处理

1. **未保存保护**：编辑批注或正文时关闭页面，弹出 Alert 确认
2. **原始链接失效**：提示用户查看"网页存档"
3. **ASR 识别错误**：校对模式允许直接修改，修改后用于导出
4. **静音阅读模式**：默认静音播放视频，完全依赖高亮逐字稿阅读（要求高亮跟随精准）
5. **视频源嗅探**：对于抖音/B站链接，系统尝试解析真实媒体地址，提供"下载原始视频/音频"
6. **加载优化**：骨架屏 (Skeleton Screen)，AI 解读流式输出
7. **极短文章**：左侧大纲自动隐藏，正文居中
8. **混合内容**：图文中嵌入视频，默认图文模式，提供"提取为视频笔记"按钮
9. **引用格式化**：任何复制操作都提供"复制为引用格式"选项，确保新闻伦理规范
10. **归档与进度**：读了一半退出自动记录阅读进度；读完手动归档

## 十二、技术实现要点

### 编辑器选型
- 推荐 **Tiptap 2.x**（基于 ProseMirror），生态丰富、可扩展性强
- 需要的 Tiptap 扩展：StarterKit, Highlight, Link, Image, Table, TaskList, CodeBlockLowlight, Placeholder, CharacterCount, FloatingMenu, BubbleMenu

### AI 集成
- 采用 SSE (Server-Sent Events) 实现流式输出
- 后端通过 Supabase Edge Function 调用 LLM API
- AI 解读结果缓存到 Postgres，避免重复请求

### 状态管理
- 推荐使用 Zustand 管理页面全局状态
- 编辑器内容使用 Tiptap 自身的状态管理
- 批注数据使用 React Context + Supabase Realtime

### 性能
- 长文章虚拟滚动
- 图片懒加载
- AI 解读预加载（进入页面后台静默生成）
- 编辑器防抖自动保存
