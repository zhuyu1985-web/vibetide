## 1. 范围与目标

### 1.1 一句话定义

> 给 vibetide 增加一个独立的"批量导入"功能，让客户上传 Excel/CSV 文件 → 浏览器 SheetJS 解析 → 字段映射（fuzzy match 自动预填 + 下拉手改）→ 试运行（前 100 行）→ 全量入库（走 A1 Writer + outlet 自动识别）。**主要用途：补齐 2025 全年历史报道数据**（main spec A2-Q7 决策的 b 双轨：tikhub 抓增量 + Excel 导入历史）。

### 1.2 范围（2-3 天工期内必交付）

- "批量导入"按钮入口（`/data-collection/sources` 顶部）+ 4 步 Dialog 流程
- SheetJS 浏览器解析（dynamic import 懒加载，主 bundle 体积不变）
- fuzzy match 字段映射（Excel 列名 → collected_items 字段；中英别名字典 ~50 条）
- 字段映射 UI：左侧 collected_items 字段 / 右侧 Excel 列下拉（默认值=fuzzy match 结果）
- 试运行扫前 100 行：显示命中字典 / 跳过重复 / 错误行 三类计数 + 5-10 条错误样本
- 全量执行：server action 分批 500 行 + Writer.writeItems 入库 + 进度反馈
- virtual source seed：每 org 一条 `__system_excel_import__`（source_type="excel_import" / category="import"）让 collection_runs / collected_items 能挂到该 source 追踪
- 错误行清单导出（CSV，客户事后修 Excel 重传）
- 测试：parse 单测 + field-mapper 单测 + bulk-import server action 集成测试

### 1.3 非目标（YAGNI / 推迟）

- ❌ Server-side 文件上传 / 临时存储（main spec Wave 2 / 大文件场景再考虑）
- ❌ 后台异步导入（Inngest job）— V1 同步导入，1-5 万行场景下可接受（同步 server action timeout 限制按需评估，可考虑分批 client→server 调用）
- ❌ 字段映射保存为模板复用（V2，客户每次重传都重选）
- ❌ Excel 模板下载（V2，客户用现有 Excel 即可）
- ❌ 导入数据手工分类 outlet_tier（A1 recognizer 已自动）
- ❌ 把 import 包装为 Adapter 注册到 registry（A2.5-Q1 决策 b：独立功能，不污染 Adapter 框架）
- ❌ 拖拽字段映射（A2.5-Q4 决策 c：仍下拉）
- ❌ 媒资库同步（main spec 全 Wave 1 不接，本 sub-spec 也不动）

### 1.4 与 main spec §4.3 对应关系

main spec §4.3 给了 Adapter 配置 schema 草稿（fileUrl / fieldMapping / defaultOutletTier / etc）。本 sub-spec：

1. **架构改为独立功能**（不走 Adapter） — Adapter 框架 ConfigFieldType 没 "file" 类型，AdapterCategory 没 "import"，硬塞会污染框架
2. **字段映射 fuzzy match 自动预填** + 下拉手改（main spec 写 drag-drop UI，调整为下拉）
3. **virtual source 替代 Adapter source**：每 org 一条 system source 让 runs / items 能挂上
4. **2025 历史数据补齐**保持 main spec A2-Q7 决策 b（双轨）

## 3. 架构设计

### 3.1 数据流

```
[浏览器]
  上传 .xlsx/.xls/.csv → FileReader → SheetJS dynamic import 解析
        ↓
  得到 { columns: string[], rows: object[] }（columns = Excel 列名，rows = 每行键值对）
        ↓
  字段映射 UI：fuzzyMatchColumns(columns) → 预填映射 → 客户审核改
        ↓
  试运行：前 100 rows + mapping → server action `previewBulkImport`
        ↓ 返回 { hitDictCount, skippedDuplicateCount, errorRows: [...] }
  客户审核 → 决定全量执行
        ↓
  全量提交：所有 rows + mapping + 默认值 → server action `executeBulkImport`
        ↓
[server]
  按 mapping 转 RawItem[]（处理 publishedAt 解析 / contentType 推断 / outletName 转 source.outletId）
        ↓
  分批 500 rows → Writer.writeItems(virtual_excel_source, items)
        ↓ Writer 自动 outlet recognizer（A1）+ fingerprint 去重 + 入 collected_items
  返回 { inserted, skipped, failed, errorClipboardCsv }
        ↓
[浏览器]
  显示完成 toast + 错误清单 CSV 下载链接
```

### 3.3 Virtual source seed

文件：`src/db/seed/excel-import-virtual-source.ts`

```ts
import { db } from "@/db";
import { collectionSources } from "@/db/schema/collection";

export const EXCEL_IMPORT_SOURCE_NAME = "__system_excel_import__";

export async function seedExcelImportVirtualSource(orgId: string) {
  await db.insert(collectionSources).values({
    organizationId: orgId,
    name: EXCEL_IMPORT_SOURCE_NAME,
    sourceType: "excel_import",
    config: {},
    targetModules: ["news_articles", "platform_content"],
```

### 4.2 fuzzy match 同义词字典

文件：`src/lib/collection/bulk-import/field-mapper.ts`

```ts
const FIELD_ALIASES: Record<string, string[]> = {
  title: ["title", "标题", "题目", "新闻标题", "报道标题", "标题列", "name", "headline"],
  content: ["content", "正文", "内容", "全文", "报道全文", "body", "text", "article"],
  summary: ["summary", "摘要", "概要", "简介", "snippet", "abstract", "description"],
  canonicalUrl: ["url", "链接", "网址", "原文链接", "原文 URL", "原文地址", "link", "source_url"],
  publishedAt: ["publishedat", "publish_time", "发布时间", "发布日期", "日期", "时间", "publish_date", "date", "publish", "发表时间"],
  outletName: ["outlet", "媒体", "媒体名", "来源", "来源媒体", "publication", "source", "source_name", "publisher", "刊发媒体"],
  outletTier: ["tier", "媒体分级", "分级", "媒体等级", "等级", "tier_level"],
  outletRegion: ["region", "区域", "地区", "省份", "城市", "省市", "area"],
  contentType: ["contenttype", "content_type", "内容类型", "类型", "类别", "type"],
};

export function matchColumns(excelColumns: string[]): Record<string, string | null> {
  const result: Record<string, string | null> = {};
  for (const [field, aliases] of Object.entries(FIELD_ALIASES)) {
    const hit = excelColumns.find((col) =>
      aliases.some((alias) => col.toLowerCase().replace(/[\s_-]/g, "").includes(alias.toLowerCase().replace(/[\s_-]/g, "")))
    );
    result[field] = hit ?? null;
  }
  return result;
}
```

匹配策略：忽略大小写 + 忽略空格/下划线/连字符 → substring includes。够用 80% 客户场景；不准时客户改下拉。

### 4.3 publishedAt 解析

文件：`src/lib/collection/bulk-import/transform.ts`

```ts
export function parseExcelPublishedAt(raw: unknown): Date | undefined {
  if (raw == null || raw === "") return undefined;

  // SheetJS 的 cellDates: true 配置时已转为 Date
  if (raw instanceof Date) return raw;

  // Excel 内部 number（自 1900 算起的天数 — 已被 SheetJS 转换，不太会遇到）
  if (typeof raw === "number") {
    if (raw > 1e12) return new Date(raw); // 毫秒时间戳
    if (raw > 1e9) return new Date(raw * 1000); // 秒时间戳
    return undefined;
  }

  if (typeof raw !== "string") return undefined;

  const s = raw.trim();
  if (!s) return undefined;

  // ISO / "2025-06-01" / "2025-06-01 14:30:00" / "2025/06/01"
  const direct = new Date(s);
  if (!isNaN(direct.getTime())) return direct;

  // 中文格式 "2025年6月1日"
  const cn = s.match(/^(\d{4})年(\d{1,2})月(\d{1,2})日$/);
  if (cn) return new Date(parseInt(cn[1]!), parseInt(cn[2]!) - 1, parseInt(cn[3]!));

  return undefined;
}
```

## 5. 试运行 + 全量执行

### 5.1 试运行（previewBulkImport server action）

输入：`{ rowCount, columns, sampleRows: 前 100 行, mapping, defaults }`

逻辑：
1. 加载 outlet 字典（loadOutletDictionaryCached - A1 已就绪）
2. 对 100 行：
   - 转 RawItem（mapping 应用）
   - parseExcelPublishedAt
   - 检查必填（title）
   - 错误行收集到 errorRows
3. 对成功行：
   - 调 recognizeOutlet 看是否命中字典 → hitDict++
   - 检查 fingerprint（select query 看 collected_items 是否已存在）→ skipped++
4. 返回 `{ hitDictCount, skippedDuplicateCount, errorRows }`

注：试运行**不写数据库**，只查询 + 分类。

### 5.2 全量执行（executeBulkImport server action）

设计为分批调用：
- client 把 rows 切 500 行一批
- 每批调一次 `executeBulkImport({ rows, mapping, defaults, batchIndex, totalBatches })`
- server action：
  1. 第一批时创建一个 `collection_run`（trigger=manual，source_id=virtual_excel_source.id）拿到 runId
  2. 后续批次复用该 runId
  3. 转 RawItem[] → 调 `writeItems({ runId, sourceId: virtualSourceId, organizationId, items, source: virtualSource })`（写入 args 对象，不是位置参数）
  4. 返回 `{ batchInserted, batchSkipped, batchFailed, errorRows, runId }`
- client 累加进度 + UI progress bar

错误行格式（CSV）：

```
行号,错误类型,详情,Excel 原始数据 JSON
5,publishedAt 解析失败,"无法识别 '二零二五年六月一日'","{...row data...}"
17,title 为空,"必填字段缺失","{...}"
```

下载方式：client 累计 error rows 后构造 Blob → URL.createObjectURL → 提供下载链接。
