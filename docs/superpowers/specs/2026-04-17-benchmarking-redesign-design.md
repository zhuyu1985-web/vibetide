# 同题漏题模块 UX 重构设计

> **日期：** 2026-04-17
> **状态：** 已确认
> **分支：** feature/genspark-redesign-phase1

---

## 1. 问题分析

当前 `/benchmarking` 模块存在以下核心 UX 问题：

1. **信息架构混乱** — 5 个平级 Tab（监控看板/同题对标/漏题筛查/预警中心/对标报告）堆在单页面，用户不知先看哪个
2. **操作流方向反了** — 当前实现是"搜索话题 → 看分析"，但业务流应该是"从我方作品出发 → 查看全网同题"
3. **没有独立详情页** — 同题对比和漏题详情都挤在列表内展开，层级太浅，信息密度过高
4. **核心概念混在一起** — "同题对比"和"漏题筛查"两个不同业务场景缺少清晰边界
5. **单体组件** — 1033 行的 `benchmarking-client.tsx` 管理过多状态，维护困难

---

## 2. 设计决策

| 决策项 | 选择 | 理由 |
|-------|------|------|
| 模块拆分 | 拆为两个独立侧边栏入口 | 业务场景不同，各有子流程，合并只增加认知负担 |
| 同题对比操作流 | 列表驱动（从我方作品出发） | 与编辑部实际工作流一致：先看自己发了什么，再对比别人 |
| 同题详情页子Tab | 保留3个（概览/列表/竞品） | 每个子功能信息量足够独立成Tab |
| 漏题列表布局 | 顶部KPI看板 + 下方列表 | 简洁直接，重心在列表操作 |
| 漏题详情布局 | 左右分栏 | 左原文右分析，对比阅读效率高 |
| 监控看板/预警/报告 | 本期去掉 | 聚焦核心功能，后续迭代再加 |

## 4. 同题对比模块

### 4.1 列表页 `/topic-compare`

**页面结构：**
- 页面标题 + 描述
- 筛选区（时间范围/发布渠道/同题状态/关键词搜索）
- 作品表格

**表格字段：**

| 列名 | 说明 | 交互 |
|------|------|------|
| 作品标题 | 标题文本，24h内标记"新" | 点击进入详情页，hover显示200字摘要 |
| 发布时间 | 首次发布时间 | 支持排序 |
| 发布渠道 | 彩色标签，最多3个+"更多" | — |
| 内容类型 | 图文/视频/直播/短视频 | 图标+文字 |
| 阅读量 | 数值缩写（1.2万） | 支持排序 |
| 同题报道数 | Badge样式，按数量分色 | 红>30 / 黄10-30 / 蓝<10 / 灰=未分析 |
| 操作 | "查看同题对比 →" | 点击进入详情 |

**筛选区：**
- 时间范围：日期区间选择器，默认近7天，最长90天
- 发布渠道：多选下拉（APP/微信/微博/抖音/视频号/网站等）
- 同题状态：单选标签（全部/已分析/未分析）
- 关键词搜索：按标题模糊搜索

**交互细节：**
- 默认按发布时间倒序
- 每页20条，支持翻页
- 传播数据使用数值缩写

### 4.2 详情页 `/topic-compare/[id]`

**页面顶部（所有子Tab共享）：**
- 面包屑导航（← 返回作品列表）
- 我方作品标题（大字）
- 发布时间 | 渠道标签组 | 传播数据摘要（阅读/点赞/评论/转发）
- 「刷新数据」按钮

**数据概览卡片（5个横排）：**

| 卡片 | 样式 |
|------|------|
| 全网报道总量 | 大数字 + 趋势对比 |
| 央级媒体报道数 | 红色背景强调 |
| 省级媒体报道数 | 蓝色背景强调 |
| 其他媒体报道数 | 灰色背景 |
| 报道时间跨度 | 最早~最新时间 |

**子Tab 1：全网报道概览（AI总结）**

AI总结面板分4个板块，每个板块独立呈现：

1. **官媒及央媒报道分析**（红色标签）
   - 央级媒体报道标题列表
   - AI提炼报道角度/切入点
   - 关键论点和引用信源
   - 每条支持「查看原文」

2. **其他媒体报道分析**（蓝色标签）
   - 按省级/都市/行业/自媒体分类
   - 差异化报道视角
   - 深度、时效、独家性评估

3. **报道亮点与创新点**（黄色标签）
   - 创新报道角度
   - 标注"我方未涉及"/"已覆盖"
   - 可借鉴策略建议

4. **整体报道总结**（灰色标签）
   - 200-300字总结
   - 热度走势 + 舆论方向 + 我方定位
   - 跟进建议（是否追加、建议角度、紧急程度）

**交互细节：**
- 流式输出，逐板块渲染
- 每个板块支持折叠/展开、独立重新生成
- 首次进入自动触发AI分析，后续使用缓存（2小时有效期）
- 底部：复制全文 / 导出文档 / 👍👎反馈
- 来源链接新窗口打开

**子Tab 2：全网报道列表**

- 筛选区：媒体级别（多选标签）/ 时间排序 / 关键词过滤 / 内容类型
- 列表/卡片视图切换（偏好持久化）
- 列表字段：报道标题 / 来源媒体+级别标签 / 报道时间 / 作者 / 摘要 / AI解读按钮 / 查看原文
- 点击「AI解读」展开/弹窗显示单篇解读（核心角度/关键信息/独有视角/写作手法/信源分析/参考价值）
- AI解读结果缓存，支持批量AI解读（一次最多5篇）
- 每页30条

**子Tab 3：竞品媒体对标**

- 按媒体级别分组展示（央级红色/省级蓝色/市级绿色）
- 每组字段：媒体名称+级别Badge / 报道标题 / 报道主体（AI提取）/ 发布时间 / 发布渠道 / AI解读 / 查看原文
- 空状态：某级别无报道时显示提示和可能原因
- 竞品AI解读：角度对比 / 时效对比 / 深度对比 / 独家内容 / 可借鉴之处 / 总评

## 5. 漏题筛查模块

### 5.1 列表页 `/missing-topics`

**顶部KPI看板（5个卡片横排）：**

| 指标 | 颜色 | 说明 |
|------|------|------|
| 今日线索总量 | 默认灰 | 今日入库线索总数 |
| 疑似漏题数 | 红色强调 | 系统筛查出的疑似漏题 |
| 已确认漏题 | 橙色 | 人工确认的漏题 |
| 已处置 | 绿色 | 已推送至三方系统 |
| 线索覆盖率 | 蓝色 | 已报道线索占总量百分比 |

**筛选区：**
- 时间范围：默认今天，支持近3天/7天
- 线索来源：多选标签（🔥社媒热榜 / 📰对标媒体 / ⚠️舆情预警）
- 漏题状态：下拉（全部/疑似漏题/已确认漏题/已排除/已覆盖）
- 排序：热度降序（默认） / 时间倒序

**线索表格字段：**

| 列名 | 说明 | 交互 |
|------|------|------|
| 线索标题 | 紧急程度图标（🔴/🟡/🟢）+ 标题 + 多源交叉标签 | 点击查看详情 |
| 线索来源 | 彩色标签（橙=社媒/蓝=对标媒体/红=舆情） | — |
| 来源详情 | 具体来源文字（微博热搜#3等） | — |
| 热度值 | 进度条+数值（0-100） | 颜色跟随数值 |
| 入库时间 | 采集入库时间 | — |
| 漏题状态 | 状态标签（绿=已覆盖/橙=疑似/红=已确认/灰=已排除） | — |
| 操作 | "查看详情 →" | 点击进入详情 |

**紧急程度判定：**
- 🔴紧急：热度>80 且 央级已报道 且 发现<6小时
- 🟡一般：热度50-80 或 省级已报道
- 🟢关注：热度<50 且 仅自媒体/社媒

**交互细节：**
- 疑似漏题行高亮（红色背景=紧急，黄色背景=一般）
- 多源交叉标签（🔗紫色，2个以上来源检测到同一事件）
- 支持批量选择后批量排除
- 默认按热度降序

### 5.2 详情页 `/missing-topics/[id]`

**顶部：** 面包屑导航 + 状态标签 + 紧急程度

**左侧面板（原文信息）：**
- 线索标题
- 来源标签组（支持多个来源标识）
- 信息网格：来源平台 / 发布时间 / 发现时间 / 当前热度（进度条） / 原文链接
- 内容摘要（抓取的正文或iframe嵌入）
- 已报道媒体列表（按级别分色展示）

**右侧面板（AI分析）：**
- 「一键AI检索全网报道」按钮
- 点击后流式输出5个板块：
  1. 官媒及央媒报道分析（同 4.2）
  2. 其他媒体报道分析
  3. 报道亮点与创新点
  4. 整体报道总结
  5. **补充报道建议**（漏题专属）：
     - 是否需要报道 + 紧急程度（立即/今日内/择时/可不报道）
     - 建议报道角度（含差异化说明、目标受众、内容形式）
     - 每个角度支持"复制为选题"
     - 关键信息点覆盖清单
     - 建议信源/采访对象
     - 风险提示（政策敏感、事实待核实等）

**底部操作区（固定在底部）：**

| 按钮 | 功能 | 后续状态 |
|------|------|---------|
| 关联已有作品 | 弹出搜索组件，在我方作品库检索关联 | → 已覆盖 |
| 确认为漏题 | 人工确认 | → 已确认漏题 |
| 排除 | 弹出理由选择（预设+自定义） | → 已排除 |
| 转为选题 | 预填信息至选题系统 | 创建选题单 |
| 推送处置 | 弹出推送配置弹窗 | → 已推送 |

**推送配置弹窗：**
- 推送通道：多选（企业微信/钉钉/采编系统/站内消息）
- 推送对象：从通讯录选择
- 紧急程度：普通/紧急/特急
- 附带AI分析：开关
- 确认推送后记录推送日志

## 9. 状态管理与缓存策略

### 筛选状态
- 列表页筛选条件在当前会话内保持（使用 URL searchParams）
- 从详情页返回列表页时恢复筛选状态（通过 URL 参数）
- 刷新页面后筛选重置为默认值

### AI 分析缓存
- 缓存 Key = `{articleId/topicId}_{reportDataHash}`（主题ID + 报道数据快照哈希）
- 有效期 2 小时，过期后用户可点击「刷新数据」手动触发重新分析
- 缓存过期时用户正在查看：继续展示旧结果，顶部提示"分析结果已过期，点击刷新获取最新数据"
- AI 解读缓存以文章 ID 为 key，有效期 2 小时，手动刷新时重新生成

### 页面导航
- 详情页无表单状态需要保护，用户可自由导航离开
- 不支持详情页之间直接切换，需返回列表页再进入下一条

## 10. 异常与空状态处理

### 空状态
| 场景 | 展示内容 |
|------|---------|
| 作品列表为空（无筛选） | 图标 + "暂无已发布作品" + "作品数据将从发布系统自动同步" |
| 作品列表筛选无结果 | "未找到匹配的作品，请调整筛选条件" |
| 线索列表为空 | 图标 + "暂无线索数据" + "系统将定时从社媒热榜、舆情系统、对标媒体采集线索" |
| 同题详情无全网报道 | "未检索到相关报道，该话题可能尚未引起广泛关注" |
| 竞品对标某级别无报道 | "该级别竞品媒体暂无同题报道" + 可能原因提示 |

### 错误状态
| 场景 | 处理 |
|------|------|
| 列表数据加载失败 | 展示错误提示 + 重试按钮 |
| AI 分析调用失败 | 自动重试 1 次，仍失败展示"分析暂时不可用，请稍后重试" + 手动重试按钮 |
| 单篇 AI 解读失败 | 该条展示错误提示，不影响其他条目 |
| 推送操作失败 | Toast 提示"推送失败：{错误原因}"，推送状态标记为失败 |

### 加载状态
- 列表页：表格区域显示 Skeleton 行（20行灰色占位），KPI 卡片显示数字 Skeleton
- 详情页：顶部信息和 KPI 卡片由 server 端渲染，AI 分析面板 client 端流式加载（SSE）
- AI 分析：逐板块流式渲染，未加载的板块显示 "正在分析..." 占位
- 用户可在 AI 分析完成前离开页面

## 11. 设计约束

- 所有 UI 文本使用中文
- 按钮无边框（项目全局约定）
- 标题不加粗，增大表格行距，max-width 居中（已有 feedback）
- Glass UI 设计系统：复用 GlassCard、StatCard 等现有组件
- 媒体级别颜色体系：央级红#E53935 / 省级蓝#1E88E5 / 市级绿#43A047 / 行业紫#8E24AA / 自媒体灰#757575
- AI输出支持流式（SSE）、缓存2小时、底部👍/👎反馈
- 紧急程度基于入库时间计算（当前时间 - discoveredAt）
- 多源交叉标签：同一事件在 2 个以上不同类型来源中被检测到时触发，每行最多 1 个此标签
- 侧边栏图标：同题对比用 `Target`（lucide），漏题筛查用 `SearchX`（lucide）
