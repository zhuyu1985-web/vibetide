# 模块一：AI 资产重构

> 定位：让沉睡的资产开口说话 — 从"被动存储"到"主动智能"
>
> 包含功能：1.1 媒资智能理解中心 | 1.2 频道顾问工坊 | 1.3 资产盘活引擎
>
> 版本：v3.0（开发实施版） | 日期：2026-03-05
>
> 说明：本文档基于 `2_模块二_AI资产重构_v2.md` 产品规格书，面向开发实施，补充完整的路由映射、数据模型、Server Action 设计和验收标准。

## 1. 模块概览

### 1.1 业务目标

让沉睡的资产开口说话，从"被动存储"到"主动智能"。具体来讲：

- 数十万级媒资从"只能存、搜不动"升级为"自动理解、秒级检索"
- 频道从"统一风格"升级为"每个频道拥有专属 AI 顾问"
- 历史素材从"用完即弃"升级为"热点关联、持续盘活"

### 1.2 目标用户

| 角色 | 使用场景 |
|------|---------|
| 媒资管理员 | 管理看板：查看 AI 理解进度、处理队列、标注纠错 |
| 一线编辑 | 通过 📦小资 间接使用：对话检索素材、获取推荐 |
| 频道负责人 | 定义和培养频道顾问、管理频道知识库 |

### 1.3 优先级与时间线

- P0 核心功能：Q1（媒资理解、知识库管理、顾问创建）
- P1 增强功能：Q1-Q2（标注纠错、资产盘活、知识源订阅）
- P2 高级功能：Q2+（A/B 测试、多语适配增强）

### 1.4 核心 AI 员工

| AI 员工 | 能力来源 | 职责 |
|---------|---------|------|
| 📦 素材管家·小资（`xiaozi`） | 1.1 媒资理解 + 1.3 资产盘活 | 看懂每条素材、主动推荐关联素材 |
| 🧠 频道顾问（`advisor`） | 1.2 频道顾问工坊 | 懂频道风格、懂垂直知识、懂受众偏好 |

AI 员工元数据定义于 `src/lib/constants.ts` 的 `EMPLOYEE_META` 对象中，`xiaozi` 对应素材管家，`advisor` 对应频道顾问。

### 1.5 模块间依赖

```
模块四 AI Team Engine ──→ 提供员工管理基础设施（员工注册、技能绑定）
       │
       ↓
┌─────────────────┐
│ 模块一 AI 资产重构 │
│                   │
│ 1.1 媒资理解     │──→ 为模块二 2.1 创作中心提供素材推荐（📦小资）
│ 1.2 频道顾问     │──→ 为模块二 2.1 创作中心提供风格把关（🧠顾问）
│ 1.3 资产盘活     │──→ 依赖模块二 2.3 热点感知引擎的输出（🔭小雷 发现热点）
└─────────────────┘
```

### 1.6 核心理念

本模块构建的不是"工具"，而是两个 AI 员工的核心能力。用户不直接使用"媒资理解引擎"或"知识库后台"，而是通过与 📦小资 对话检索素材、通过 🧠顾问 获得风格把控。

### 2.6 验收标准

**AC-1.1.01：资产理解状态列表**
- GIVEN 用户进入媒资智能理解中心页面（`/asset-intelligence`）
- WHEN 页面加载完成
- THEN 显示所有媒资的理解状态列表，包含"待处理"、"处理中"、"已完成"、"失败"四种状态

**AC-1.1.02：视频理解详情**
- GIVEN 一条视频素材已完成 AI 理解
- WHEN 用户查看详情（自动理解 Tab）
- THEN 显示时间轴标签、场景分割、ASR 转写、OCR 提取、NLU 摘要、人物标记

**AC-1.1.03：语义搜索**
- GIVEN 用户在团队工作台 @小资 或在语义搜索输入框中
- WHEN 输入自然语言查询（如"养老金改革的采访画面"）
- THEN 返回匹配的视频片段列表，按匹配度（relevanceScore）降序排序

**AC-1.1.04：标注纠错**
- GIVEN AI 自动标注结果有误
- WHEN 管理员修正标签
- THEN 修正数据记录并标记来源为 `human_correct`，原 AI 标注保留历史记录

**AC-1.1.05：处理队列看板**
- GIVEN 有多条资产在处理队列中
- WHEN 用户查看处理队列 Tab
- THEN 显示排队中/处理中/已完成/失败的统计数和详细列表，处理中的显示实时进度

**AC-1.1.06：知识图谱可视化**
- GIVEN 多条资产已完成理解并生成知识节点
- WHEN 用户切换到自动关联 Tab
- THEN 显示知识图谱节点关联网络，点击节点高亮关联边和相邻节点
