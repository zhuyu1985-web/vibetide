## 模块依赖关系

```
                    ┌─────────────────────────────┐
                    │   模块四 · AI 团队引擎（基座）  │
                    │                              │
                    │  4.1 员工管理与团队组建 ✅      │
                    │  4.2 可视化团队编排    (Q4)    │
                    │  4.3 自学习进化基座    (Q4)    │
                    └──────────┬──────────────────┘
                               │ 被所有模块依赖
              ┌────────────────┼────────────────┐
              ↓                ↓                ↓
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│ 模块一           │  │ 模块二           │  │ 模块三           │
│ AI 资产重构      │  │ 智创生产         │  │ 全渠道传播       │
│                 │  │                 │  │                 │
│ 1.1 媒资理解  ──→│──→ 2.1 创作中心    │  │ 3.1 矩阵管理     │
│ 1.2 频道顾问  ──→│──→ 2.1 风格把关    │  │ 3.2 旗舰 APP ←──│── 1.2 知识库
│ 1.3 资产盘活  ←──│── 2.3 热点联动     │  │ 3.3 精品率  ←───│── 2.2 竞品数据
│                 │  │                 │  │                 │
│ Q1-Q2           │  │ Q1-Q3           │  │ Q3-Q4           │
└─────────────────┘  └─────────────────┘  └─────────────────┘
                                                    │
                                                    ↓ 数据回流
                                            4.3 自学习进化基座
```

**依赖方向：**
- 模块四是全平台基座，所有模块依赖其 AI 员工管理和团队协作能力
- 模块一为模块二提供素材管理（📦小资）和频道风格（🧠顾问）能力
- 模块二的内容产出是模块三的输入
- 模块三的传播效果数据回流至模块四的自学习基座

## 文档约定

### 功能编号规则

功能需求使用 `F{模块}.{子模块}.{序号}` 格式编号：

| 示例 | 含义 |
|------|------|
| F1.1.01 | 模块一 → 子模块 1.1 → 第 1 个功能 |
| F2.3.05 | 模块二 → 子模块 2.3 → 第 5 个功能 |
| F4.1.10 | 模块四 → 子模块 4.1 → 第 10 个功能 |

### 数据源状态标记

| 标记 | 含义 |
|------|------|
| ✅ DB | 已连接数据库，使用 DAL + Server Actions |
| Mock | UI 已实现，数据来自 `src/data/` 静态文件 |
| 未开始 | 无页面、无数据源 |
