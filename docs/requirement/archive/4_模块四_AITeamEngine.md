# 模块四 · AI Team Engine — AI智能员工团队引擎

> 本模块为 Vibe Media 数智全媒平台的核心基座，定义AI智能员工的构建、管理、
> 协作与进化机制。所有上层业务功能（资产管理、内容生产、渠道分发）的AI能力
> 均通过本模块的AI员工团队来承载和交付。
>
> 版本：v2.0 ｜ 合并原4.1 OpenClaw + 原2.5 多Agent编排画布 + 新增4.3 自学习基座

---

## 设计哲学

### 从 Claude Code Agent Team 获得的启发

Claude Code 的 Agent Team 模式证明了一个关键洞察：**当AI以"有角色、有职责、能协作的团队成员"身份出现时，用户的理解成本和协作效率都大幅优于"调用一个个API/工具"的模式。**

我们将这一理念深度适配到传媒行业：

| Claude Code Agent Team | Vibe Media AI Employee Team |
|------------------------|----------------------------|
| Lead Agent 分解编程任务 | Team Hub 分解内容生产目标 |
| Sub-agent 专注特定代码领域 | AI员工专注特定媒体工作领域 |
| Agent 间通过上下文传递协作 | AI员工间通过任务流+工作物传递协作 |
| 人类开发者在关键节点review | 编辑在关键节点审核决策 |
| Agent 记住项目上下文和偏好 | AI员工记住频道风格和用户习惯 |

### 三层架构原则

```
┌─────────────────────────────────────────────────┐
│  第一层：感知与表达（用户看到的）                   │
│                                                  │
│  AI员工 = 有名字 + 有头像 + 有性格 + 会说人话       │
│  用户通过对话、审批、目标输入与AI员工协作            │
│                                                  │
├─────────────────────────────────────────────────┤
│  第二层：协作与编排（团队如何工作）                  │
│                                                  │
│  任务分解 → 员工分配 → 工作物传递 → 人机决策         │
│  支持自动编排 / 可视化编排 / 模板编排三种方式         │
│                                                  │
├─────────────────────────────────────────────────┤
│  第三层：能力与进化（员工凭什么能干活）               │
│                                                  │
│  Skill技能绑定 + 知识库绑定 + 模型调优 + 数据沉淀    │
│  员工能力随使用不断进化                              │
│                                                  │
└─────────────────────────────────────────────────┘
```

### 核心概念模型

#### 4.1.1 AI员工（AI Employee）

一个AI员工是平台中最核心的交互实体，它由以下要素构成：

**authority_level（权限等级）定义：**

| 等级 | 名称 | 说明 | 举例 |
|------|------|------|------|
| observer | 观察者 | 仅能查看和汇报，不能执行动作 | 数据看板只读 |
| advisor | 顾问 | 可分析和建议，执行需人工确认 | 热点推荐需编辑确认 |
| executor | 执行者 | 可在预设范围内自主执行 | 批量格式转换自动完成 |
| coordinator | 协调者 | 可指挥其他员工协作 | Team Lead分配子任务 |

#### 4.1.2 Skill（技能）

技能是AI员工的"能力组件"，一个员工可绑定多个技能：

```json
{
  "skill_id": "skill_trending_scan",
  "name": "全网热点扫描",
  "version": "1.3.0",
  "category": "perception",
  "description": "扫描30+平台的热点数据，实时计算热度指数",
  "input_schema": {
    "platforms": "string[] — 要扫描的平台列表",
    "time_range": "string — 时间范围",
    "keywords_filter": "string[] — 关键词过滤"
  },
  "output_schema": {
    "trending_list": "Trending[] — 热点列表，按热度排序",
    "scan_stats": "ScanStats — 扫描统计"
  },
  "runtime": {
    "type": "api_chain",
    "avg_latency_ms": 3200,
    "max_concurrency": 5,
    "model_dependency": "claude-sonnet-4-5 + custom_trending_model_v2"
  },
  "compatible_roles": ["trending_scout", "content_strategist"]
}
```

**技能分类体系：**

| 类别 | 代码 | 包含技能示例 |
|------|------|------------|
| 感知类 | perception | 热点扫描、舆情监测、竞品追踪、数据采集 |
| 分析类 | analysis | 热度评估、情感分析、竞品对标、效果分析 |
| 生成类 | generation | 文案生成、图片生成、视频脚本、标题优化 |
| 制作类 | production | 视频剪辑、音频合成、图文排版、数字人播报 |
| 管理类 | management | 任务分解、进度追踪、质量评估、发布调度 |
| 知识类 | knowledge | 知识图谱查询、语义检索、FAQ问答、风格参考 |

## 预置AI员工详细设计

### 员工1：🔭 热点猎手·小雷

| 维度 | 设计 |
|------|------|
| 角色定义 | 7×24小时全网热点监控与预警专家 |
| 性格特质 | 敏锐、果断、偶尔激动（发现大热点时） |
| 绑定技能 | 全网热点扫描、热度评分、趋势预判、竞品响应追踪 |
| 典型输出 | 热点预警卡片（含热度/趋势/竞品/建议动作） |
| 权限等级 | advisor — 发现热点可自动通知，但启动团队响应需人工确认 |
| 主动行为 | 持续扫描热点，P0级自动推送，P1级整点汇总，P2级日报 |
| 与谁协作 | 发现热点后自动@选题策划师预生成角度 |
| 可学习项 | 用户关注领域偏好、热点判断阈值、推送时间偏好 |

### 员工2：💡 选题策划师·小策

| 维度 | 设计 |
|------|------|
| 角色定义 | 选题角度推荐与策划辅助专家 |
| 性格特质 | 创意丰富、条理清晰、善于发现独特视角 |
| 绑定技能 | 角度生成、匹配度评估、评论洞察分析、竞品选题gap分析 |
| 典型输出 | 选题角度卡片组（含角度/匹配度/理由/参考素材） |
| 权限等级 | advisor — 角度建议需人工选择确认 |
| 主动行为 | 收到热点猎手推送时自动生成角度，每日8:00推送选题建议 |
| 与谁协作 | 上游接收热点猎手推送，下游将确认角度传递给内容创作师 |
| 可学习项 | 用户角度偏好、标题风格、选题规律 |

### 员工3：📦 素材管家·小资

| 维度 | 设计 |
|------|------|
| 角色定义 | 台内媒资管理与智能推荐专家 |
| 性格特质 | 细心、记忆力强、分类严谨 |
| 绑定技能 | 媒资语义检索、热点-资产关联、知识图谱查询、资产盘活推荐 |
| 典型输出 | 素材推荐清单（含匹配度/预览/历史使用记录） |
| 权限等级 | executor — 素材检索和推送可自主执行 |
| 主动行为 | 监听团队创作任务，自动预备相关素材 |
| 与谁协作 | 为创作师和视频制片人准备素材 |
| 可学习项 | 用户常用素材偏好、素材质量标准 |

### 员工4：✍️ 内容创作师·小文

| 维度 | 设计 |
|------|------|

| 典型输出 | 图文内容（标题+正文+配图+摘要） |
| 权限等级 | executor — 生成初稿自动进行，终稿需人工确认 |
| 主动行为 | 收到选题确认后自动开始创作 |
| 与谁协作 | 从选题策划师接收角度，从素材管家接收素材，完成后交审核官 |
| 可学习项 | 写作风格、标题偏好、用词禁忌、文章长度偏好 |

| 与谁协作 | 从素材管家接收视频素材，完成后交审核官 |
| 可学习项 | 剪辑风格、时长偏好、字幕样式、配乐倾向 |

### 员工6：🔍 质量审核官·小审

| 维度 | 设计 |
|------|------|

| 典型输出 | 审核报告（通过/问题清单/修改建议/爆品预测分） |
| 权限等级 | advisor — 审核结论供参考，最终发布需人工确认 |
| 主动行为 | 内容提交后立即审核，发现敏感问题即时预警 |
| 与谁协作 | 审核创作师和制片人的产出，通过后交渠道运营师 |

### 员工7：📡 渠道运营师·小发

| 维度 | 设计 |
|------|------|

| 绑定技能 | 渠道格式适配、最佳时间推荐、批量发布、互动管理、SEO优化 |
| 典型输出 | 多渠道发布计划+执行结果（含各渠道格式适配稿） |
| 权限等级 | executor — 非敏感内容可自主发布，敏感内容需审批 |
| 主动行为 | 内容审核通过后自动生成最佳发布时间建议 |
| 与谁协作 | 从审核官接收通过内容，发布后将数据推送给数据分析师 |
| 可学习项 | 各渠道最佳发布时间、标题优化策略、互动话术 |

### 员工8：📊 数据分析师·小数

| 维度 | 设计 |
|------|------|
| 角色定义 | 传播效果分析与策略优化专家 |
| 性格特质 | 理性、数据敏感、善于发现规律 |
| 绑定技能 | 全渠道数据聚合、效果评估、竞品对标分析、趋势预测、优化建议生成 |
| 典型输出 | 效果报告（数据+洞察+优化建议） |
| 权限等级 | advisor — 分析和建议，优化策略需人工确认 |
| 主动行为 | 内容发布2h/24h/72h自动推送效果报告，异常数据实时预警 |
| 与谁协作 | 从运营师获取发布数据，分析结果反馈给选题策划师形成闭环 |
| 可学习项 | 用户关注的KPI偏好、报告格式偏好、预警阈值 |

| 可学习项 | 频道风格演化、用户对风格建议的采纳规律 |

### 关键技术决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| AI员工运行时 | 长驻进程 + 事件驱动 | 需要7×24在线+主动行为 |
| 团队通信 | 消息队列（工作物传递）+ WebSocket（实时推送） | 异步协作 + 实时通知 |
| 记忆存储 | 向量数据库（长期偏好）+ Redis（短期上下文） | 兼顾学习深度和响应速度 |
| 目标解析 | LLM链式推理 + 工作流模板匹配 | 自然语言理解 + 可靠执行 |
| 画布渲染（4.2） | React Flow | 成熟的拖拽画布方案 |
