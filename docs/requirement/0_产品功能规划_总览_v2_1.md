## 核心理念变革：从"AI工具"到"AI团队"

### 场景痛点

一个典型的地方电视台新媒体部门，仅有2-3名编辑，却需要完成：

```
热点监控（7×24h） → 选题策划 → 素材采集 → 多种媒介内容创作
    → 合规审核 → 6+渠道适配发布 → 数据分析 → 竞品对标 → 策略优化
```

**这是一个至少需要15-20人的工作量。2个人根本做不过来。**

### 解决方案：AI智能员工团队（AI Employee Team）

受 Claude Code Agent Team 理念启发，我们不再让用户面对一堆"AI工具"和抽象的"Agent"，而是为用户构建**一支可配置、可协调、可进化的AI智能员工团队**。

**核心范式转换：**

| 旧范式（AI工具） | 新范式（AI团队） |
|-----------------|-----------------|
| 用户学习使用各种AI工具 | 用户像管理团队一样分配任务 |
| "我要用哪个功能？" | "这件事交给谁做？" |
| 功能散落在各个菜单里 | AI员工主动汇报工作、请示决策 |
| 用户需要理解技术概念 | 用户只需要表达业务目标 |
| 工具没有记忆和成长 | AI员工越干越懂你的要求 |

### 产品Slogan

> **"两个编辑 + 一支AI团队 = 一个全能新媒体军团"**

## AI智能员工体系设计

### 预置AI员工角色（开箱即用）

| 角色名称 | 职责定位 | 核心能力 | 对应原功能模块 |
|---------|---------|---------|--------------|
| 🔭 **热点猎手** | 7×24小时全网热点监控与预警 | 30+平台扫描、热度评估、分级推送、趋势预判 | 2.3 热点感知引擎 |
| 💡 **选题策划师** | 选题角度推荐与策划辅助 | 角度生成、匹配度评估、评论洞察、竞品选题gap分析 | 2.3 角度推荐 + 2.2 漏题分析 |
| 📦 **素材管家** | 台内资产管理与智能推荐 | 媒资语义检索、热点-资产关联、历史素材推荐、知识图谱查询 | 1.1 媒资理解 + 1.3 资产盘活 |
| ✍️ **内容创作师** | 多媒介内容自动/辅助生产 | AI写稿、AI配图、AI视频脚本、AI配音、数字人播报 | 2.1 AIGC车间 + 2.4 批量生产 |
| 🎬 **视频制片人** | 视频类内容专项生产 | 大屏转小屏、赛事拆条、会议集锦、短视频生产 | 2.4 视频适配 + 2.6 节赛会展 |
| 🔍 **质量审核官** | 内容合规审核与质量评估 | 敏感词检测、事实核查、版权检测、爆品指数预测 | 3.1 审核 + 3.3 质检 |
| 📡 **渠道运营师** | 多渠道适配分发与运营 | 渠道格式适配、最佳时间推荐、发布执行、互动管理 | 3.1 矩阵管理 |
| 📊 **数据分析师** | 传播效果分析与策略优化 | 全渠道数据聚合、效果评估、竞品对标报告、优化建议 | 3.1 数据看板 + 2.2 竞品对标 |
| 🧠 **频道顾问**（按频道配置） | 特定频道的风格把控与知识专家 | 栏目基因理解、风格约束、垂直知识、个性化创作 | 1.2 栏目知识库 |

### AI员工 vs 传统Agent的区别

```
传统Agent:                          AI智能员工:
┌──────────────┐                   ┌──────────────────────────────┐
│ agent_id: 007 │                   │ 👤 热点猎手·小雷              │
│ type: trending │                  │                              │
│ status: online │                  │ "老板，发现一条必追热点！      │
│ calls: 12,847  │                  │  央视和人民日报都已响应，       │
│ latency: 230ms │                  │  建议我们15分钟内出稿。        │
│                │                  │  我已准备好3个切入角度，       │
│ [调用API]      │                  │  要安排内容创作师开工吗？"     │
│                │                  │                              │
│ ← 用户看不懂   │                  │ [查看角度] [安排创作] [稍后]   │
└──────────────┘                   └──────────────────────────────┘
```

**关键设计原则：**

1. **拟人化交互**：每个AI员工有名字、头像、职责描述，用自然语言汇报工作和请示决策
2. **主动沟通**：AI员工不是被动等待调用，而是主动发现问题、推送建议、请求审批
3. **团队协作**：AI员工之间自动协作（热点猎手发现热点→选题策划师生成角度→内容创作师开始生产），用户看到的是团队工作流
4. **可配置**：用户可以调整每个AI员工的工作偏好、汇报频率、自主决策权限
5. **可扩展**：基于预置角色，用户可以创建自定义AI员工，组合不同Skill

## AI员工与用户的协作模式

### 模式一：目标驱动模式（最核心）

```
用户输入目标 ────────────────────────────────────────────
    │
    │ "针对两会政府工作报告，今天出3条图文+2条短视频，
    │  分发微信和抖音，要求突出民生视角"
    │
    ↓
AI Team自动编排 ──────────────────────────────────────────
    │
    ├→ 🔭热点猎手：确认热度98，竞品14/20已响应，窗口紧迫
    ├→ 💡选题策划师：生成4个民生角度，推荐TOP2
    ├→ 📦素材管家：关联台内历史素材23条，准备就绪
    │      ↓（等待用户确认角度）
    │
    ├→ ✍️内容创作师：开始生产图文3篇
    ├→ 🎬视频制片人：开始生产短视频2条
    │      ↓（等待用户审核内容）
    │
    ├→ 🔍质量审核官：合规检测通过
    ├→ 📡渠道运营师：微信10:00/抖音10:30发布
    │      ↓
    └→ 📊数据分析师：2小时后推送首波效果报告
```

### 模式二：单员工对话模式

```
用户直接@某个AI员工进行对话交互：

用户：@数据分析师 上周微信公众号表现怎么样？和抖音比呢？

📊数据分析师：上周微信公众号核心数据如下：
    - 发文23篇，总阅读58.6万，环比↑12%
    - 互动量1.2万，互动率2.1%（高于行业均值1.5%）
    - vs 抖音：阅读量微信仅为抖音的32%，但互动率高出40%
    - 建议：微信侧重深度+互动内容，抖音侧重流量型快讯
    需要我生成详细的对比报告吗？
```

## CCBN展会核心叙事（v2.0）

**主题：** "给你2个编辑 + 1支AI团队，造一个全能新媒体军团"

**演示流程：**

1. **开场**：展示一个仅有2人的新媒体团队工作台，引出人力困境
2. **AI团队上线**：一键激活AI员工团队，展示每个员工的角色卡片和能力介绍
3. **实战演示**：以两会报道为例，展示用户输入目标→AI团队自动编排→各员工协作→内容产出→多渠道分发的完整流程
4. **人机协同**：演示关键节点人工介入（审核、调整角度、修改内容），体现"AI不是替代人，是赋能人"
5. **效果呈现**：展示AI团队运行1个月后的数据——产出效率提升300%，响应时效从45分钟到8分钟
