# 模块四：AI 团队引擎（AI Team Engine）

> 本模块为 Vibe Media 数智全媒平台的核心基座，定义 AI 智能员工的构建、管理、协作与进化机制。所有上层业务功能（资产管理、内容生产、渠道分发）的 AI 能力均通过本模块的 AI 员工团队来承载和交付。
>
> 版本：v3.0 | 基于已完成的数据库迁移和前端页面连接整理

---

## 1. 模块概览

### 1.1 业务目标

构建 AI 智能员工团队的管理、协作与进化基座，使 AI 以「有角色、有职责、能协作的团队成员」身份呈现，让传媒机构能够像管理真人团队一样管理 AI 员工团队。

### 1.2 目标用户

| 角色 | 使用场景 |
|------|---------|
| **平台管理员** | 配置 AI 员工能力、权限、知识库，管理技能库 |
| **编辑主任** | 组建内容生产团队，配置协作规则和升级策略 |
| **一线编辑** | 使用 AI 团队完成日常内容生产，与 AI 员工对话协作 |

### 1.3 优先级与时间线

| 子模块 | 优先级 | 时间线 | 状态 |
|--------|--------|--------|------|
| 4.1 AI 员工管理与团队组建 | P0 | Q1-Q2 | **已完成 DB 迁移** |
| 4.2 可视化团队编排 | P2 | Q4 | 规划中 |
| 4.3 自学习进化基座 | P2 | Q4 | 规划中 |

### 1.5 模块间依赖

```
模块一（智创生产）  ──依赖──→  4.1 AI 员工系统（编排内容生产流程）
模块二（AI 资产重构）──依赖──→  4.1 AI 员工系统（实例化素材管家和频道顾问）
模块三（全渠道传播）──依赖──→  4.1 消息系统（驱动审核/发布/分析流程）
4.3 自学习基座    ──依赖──→  所有模块的行为数据
```

### 1.6 设计哲学

从 Claude Code Agent Team 理念启发，将 AI 以「有角色、有职责、能协作的团队成员」身份呈现：

| Claude Code Agent Team | Vibe Media AI Employee Team |
|------------------------|----------------------------|
| Lead Agent 分解编程任务 | Team Hub 分解内容生产目标 |
| Sub-agent 专注特定代码领域 | AI 员工专注特定媒体工作领域 |
| Agent 间通过上下文传递协作 | AI 员工间通过任务流+工作物传递协作 |
| 人类开发者在关键节点 review | 编辑在关键节点审核决策 |
| Agent 记住项目上下文和偏好 | AI 员工记住频道风格和用户习惯 |

#### DAL 文件（数据访问层）

所有 DAL 函数为只读查询，返回 `src/lib/types.ts` 中定义的 UI 类型：

**`src/lib/dal/employees.ts`**
- `getEmployees(): Promise<AIEmployee[]>` -- 获取所有 AI 员工列表及其绑定技能
- `getEmployee(slug: string): Promise<AIEmployee | undefined>` -- 按 slug 获取单个员工及技能
- `getEmployeeFullProfile(slug: string): Promise<EmployeeFullProfile | undefined>` -- 获取员工完整档案（含知识库、工作偏好、权限配置、学习模式等）

**`src/lib/dal/teams.ts`**
- `getTeams(): Promise<Team[]>` -- 获取所有团队列表及成员概要
- `getTeam(id: string): Promise<Team | undefined>` -- 按 ID 获取单个团队
- `getTeamWithMembers(teamId: string): Promise<TeamWithMembers | undefined>` -- 获取团队详情及所有成员的完整 AI 员工信息
- `getWorkflowTemplates()` -- 获取所有工作流模板

**`src/lib/dal/skills.ts`**
- `getSkills(category?: SkillCategory): Promise<Skill[]>` -- 获取技能库，支持按分类过滤
- `getSkillsNotBoundToEmployee(employeeId: string): Promise<Skill[]>` -- 获取尚未绑定到指定员工的技能列表

**`src/lib/dal/knowledge-bases.ts`**
- `getKnowledgeBases(): Promise<KnowledgeBaseInfo[]>` -- 获取所有知识库
- `getEmployeeKnowledgeBases(employeeId: string): Promise<KnowledgeBaseInfo[]>` -- 获取指定员工已绑定的知识库

**`src/lib/dal/workflows.ts`**
- `getWorkflows(): Promise<WorkflowInstance[]>` -- 获取所有工作流实例及步骤详情
- `getWorkflow(id: string): Promise<WorkflowInstance | undefined>` -- 获取单个工作流实例

**`src/lib/dal/messages.ts`**
- `getTeamMessages(teamId?: string): Promise<TeamMessage[]>` -- 获取团队消息流，支持按团队过滤

**`src/lib/dal/auth.ts`**
- `getCurrentUserOrg(): Promise<string | null>` -- 获取当前登录用户所属组织 ID

#### Server Actions（服务端操作）

所有 Server Actions 标记为 `"use server"`，执行前均通过 `requireAuth()` 鉴权，操作后调用 `revalidatePath()` 刷新缓存。

**`src/app/actions/employees.ts`**
- `createEmployee(data)` -- 创建自定义 AI 员工，`isPreset` 标记为 0
- `updateEmployeeStatus(employeeId, status, currentTask?)` -- 更新员工状态（working/idle/learning/reviewing）
- `bindSkillToEmployee(employeeId, skillId, level?)` -- 为员工绑定技能，默认熟练度 50
- `unbindSkillFromEmployee(employeeId, skillId)` -- 解除员工的技能绑定
- `updateEmployeeProfile(employeeId, data)` -- 更新员工基本信息（名称、头衔、格言、权限等级）
- `updateWorkPreferences(employeeId, prefs)` -- 更新工作偏好（主动性、汇报频率、沟通风格等）
- `updateAuthorityLevel(employeeId, level)` -- 更新权限等级（observer/advisor/executor/coordinator）
- `updateAutoActions(employeeId, autoActions, needApprovalActions)` -- 更新自主行动和需审批行动列表
- `updateSkillLevel(employeeId, skillId, level)` -- 更新员工某项技能的熟练度
- `cloneEmployee(sourceId, newSlug, newNickname)` -- 克隆现有员工（含技能绑定），生成非预置副本
- `deleteEmployee(employeeId)` -- 删除自定义员工（预置员工禁止删除）

**`src/app/actions/teams.ts`**
- `createTeam(data)` -- 创建团队，含 AI 成员、人类成员和协作规则
- `addTeamMember(teamId, member)` -- 向团队添加成员（AI 或人类）
- `removeTeamMember(memberId)` -- 从团队移除成员
- `deleteTeam(teamId)` -- 删除团队（级联删除成员）
- `updateTeam(teamId, data)` -- 更新团队名称或场景
- `updateTeamRules(teamId, rules)` -- 更新团队协作规则（审批要求、汇报频率、敏感话题）

**`src/app/actions/workflows.ts`**
- `createWorkflowInstance(data)` -- 创建工作流实例及其步骤
- `updateWorkflowStepStatus(stepId, status, progress, output?)` -- 更新工作流步骤状态和进度

**`src/app/actions/messages.ts`**
- `sendTeamMessage(data)` -- 发送团队消息（支持 AI 或人类发送，含附件和操作按钮）

### 2.4 AI 员工集成

#### 员工状态管理

Team Hub 实时显示所有 AI 员工的工作状态：

| 状态 | 含义 | 触发条件 |
|------|------|---------|
| `working` | 忙碌 | 正在执行任务 |
| `idle` | 空闲 | 等待分配任务 |
| `learning` | 学习中 | 正在从反馈数据中学习优化 |
| `reviewing` | 审核中 | 正在审核内容（特指小审） |

#### 标准工作流流水线

8 步标准内容生产流程（定义于 `src/lib/constants.ts` 的 `WORKFLOW_STEPS`）：

```
热点监控(小雷) → 选题策划(小策) → 素材准备(小资) → 内容创作(小文)
→ 视频制作(小剪) → 质量审核(小审) → 渠道发布(小发) → 数据分析(小数)
```

每个步骤对应一个 `workflow_steps` 记录，状态流转为：`pending` -> `active` -> `completed`（或 `skipped`）。

#### 团队动态消息流

Activity Feed 展示团队消息（`team_messages` 表），支持四种消息类型：

- **alert** -- 预警通知（如热点预警）
- **decision_request** -- 决策请求（如选题角度确认）
- **status_update** -- 状态更新（如创作进度汇报）
- **work_output** -- 工作产出（如初稿完成通知）
