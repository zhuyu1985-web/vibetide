## 1. 模块概览

| 项 | 说明 |

## 2. 子模块 3.1：全媒体传播矩阵管理平台

### 2.4 AI 员工集成

| AI 员工 | 集成方式 | 触发条件 | 产出 |
|---------|----------|----------|------|
| 📡小发 (xiaofa) | 审核通过后自动生成多渠道适配版本 | 🔍小审 审核通过事件 | 6+ 渠道适配版本 + 发布时间推荐 |
| 📡小发 (xiaofa) | 执行发布任务 | 用户确认 / 条件触发 | 多渠道发布结果 |
| 🔍小审 (xiaoshen) | 内容提交后自动审核 | 模块二内容产出完成 | 通过/驳回/升级 + 问题标注 |
| 📊小数 (xiaoshu) | 发布后效果监控 | 发布 2 小时后自动触发 | 首波效果报告 → 团队消息流 |
| 📊小数 (xiaoshu) | 周期性报告 | 每周/每月定时 | 周报/月报自动推送 |

### 2.5 验收标准

```gherkin
GIVEN 内容通过 🔍小审 审核
WHEN 📡小发 生成适配版本
THEN 用户看到所有渠道的预览并可逐条微调

GIVEN 📡小发 执行发布
WHEN 发布 2 小时后
THEN 📊小数 自动推送各渠道效果数据到团队消息流

GIVEN 📊小数 发现数据异常
WHEN 负面评论激增或阅读量暴跌
THEN 自动推送预警到团队消息流，标记严重等级

GIVEN 用户在发布台查看排期
WHEN 切换日历视图
THEN 所有渠道发布计划以日历形式展示，可拖拽调整

GIVEN 用户配置条件触发规则
WHEN 条件满足（如热度达标）
THEN 📡小发 自动执行发布流程
```
