## 1. 模块概览

### 1.1 业务目标

AI 员工团队驱动的全链路内容生产。用户在本模块中的所有操作本质上都是在与 AI 员工团队协作完成任务——从热点发现、选题策划、内容创作、质量审核到渠道分发，覆盖内容生产全生命周期。

### 1.2 目标用户

| 角色 | 使用场景 |
|------|----------|
| 一线编辑、记者 | 超级创作中心（2.1）、热点看板（2.3） |
| 新媒体运营 | 超级创作中心（2.1）、批量产线（2.4） |
| 编辑主任、频道总监 | 竞品对标（2.2）、热点感知（2.3） |
| 体育编辑、大型活动报道团队 | 节赛会展自动产线（2.5） |

### 1.3 优先级与时间线

| 编号 | 子模块 | 核心 AI 员工 | 优先级 | 季度 |
|------|--------|-------------|--------|------|
| 2.1 | 超级个体创作中心 | ✍️小文 + 🎬小剪 + 📦小资 + 🧠顾问 + 🔍小审 | P0 | Q2 |
| 2.2 | 竞品对标系统 | 📊小数 | P0 | Q2 |
| 2.3 | 热点感知引擎 | 🔭小雷 + 💡小策 + 📦小资 | P0 | Q1 |
| 2.4 | 批量生产线 | ✍️小文 + 🎬小剪 + 📡小发 + 🔍小审 | P1 | Q3 |
| 2.5 | 节赛会展自动产线 | 🎬小剪（赛事模式）+ 📊小数 + 📡小发 | P1 | Q3 |

### 2.1.4 验收标准

- **GIVEN** 用户在创作中心输入创作目标（如"围绕 AI 手机发布，产出图文深度报道和 60 秒短视频"） **WHEN** 提交目标 **THEN** AI 团队自动分解任务（✍️小文 负责图文、🎬小剪 负责视频），并开始协作
- **GIVEN** ✍️小文 完成初稿 **WHEN** 用户在对话区 @小文 发出修改指令（如"第二段太硬了，换成讲故事的方式"） **THEN** 小文 按指令修改并生成新版本
- **GIVEN** 创作进行中 **WHEN** 📦小资 发现匹配素材 **THEN** 素材推荐面板实时更新，显示匹配度和素材详情
- **GIVEN** 用户选择多媒介切换（图文+视频） **WHEN** ✍️小文 撰写图文的同时 **THEN** 🎬小剪 并行生成视频脚本
- **GIVEN** 内容通过 🔍小审 审核 **WHEN** 用户确认发布 **THEN** 📡小发 自动适配各渠道格式

### 6.4 验收标准

- **GIVEN** 用户预设赛事信息并启动 🎬小剪 赛事模式 **WHEN** 检测到进球/扣篮 **THEN** 3 分钟内完成拆条 → 配字幕 → 适配格式 → 自动分发
- **GIVEN** 会议直播进行中 **WHEN** AI 实时转录并检测到金句 **THEN** 自动提取金句并生成金句卡片
- **GIVEN** 展览活动进行中 **WHEN** 记者完成某展位探访 **THEN** AI 自动生成产品卡摘要
- **GIVEN** 赛事/会议结束 **WHEN** 📊小数 生成数据报告 **THEN** 展示内容产出数量、发布数量和总浏览量

### 8.5 页面重构要求

当前所有 6 个主页面均为 `"use client"` 单文件模式（在 `page.tsx` 中直接标记 `"use client"` 并导入 Mock 数据）。迁移时需重构为标准 server/client 拆分模式：

```
// 迁移前（当前状态）
src/app/(dashboard)/inspiration/
└── page.tsx          ← "use client", 直接导入 Mock 数据

// 迁移后（目标状态）
src/app/(dashboard)/inspiration/
├── page.tsx               ← Server Component, 调用 DAL 获取真实数据
└── inspiration-client.tsx ← Client Component, 接收 props 处理交互
```
