## 1 全部资源首页

**页面定位：** 智能媒资模块的门户首页，提供全局搜索入口、内容推荐、宣传日历和专题入口，帮助用户快速发现和定位资源。

**提示词：**

```
设计一个媒体资源管理平台的首页门户页面。

页面整体布局：
- 顶部为全局搜索栏，占据页面宽度的约60%居中显示，搜索栏左侧有搜索图标，右侧有"意图检索"开关按钮，支持切换普通搜索和AI语义搜索模式。搜索栏下方有一排快捷标签（热门搜索词）。
- 搜索栏下方分为左右两栏布局：
  - 左侧主内容区（约70%宽度）：
    - "最新推荐"区块：横向滚动卡片列表，每个卡片展示资源封面图、标题、创建时间，最多展示8-12个。
    - "热门推荐"区块：同样横向滚动卡片列表，按热度排序。
  - 右侧辅助区（约30%宽度）：
    - "宣传日历"模块：显示当月日历，特殊日期（纪念日、宣传日）有标记高亮，点击日期展示关联的资源和专题。
    - "事件专题"入口：卡片式展示当前热门事件专题。
    - "人物专题"入口：卡片式展示重要人物专题。

交互说明：
- 搜索栏输入内容后回车或点击搜索，跳转到搜索结果页面。
- 意图检索模式下，搜索栏支持上传图片进行以图搜图。
- 卡片点击进入资源详情页。
- 日历中标记的日期可点击，展开该日期关联的资源列表。

视觉风格：
- 整体风格简洁专业，以白色为主背景，蓝色为主色调。
- 搜索栏突出展示，作为页面视觉焦点。
- 卡片带有圆角和轻微阴影，封面图为16:9比例。
```

## 2 个人库页面

**页面定位：** 用户个人素材管理空间，支持上传、管理、查看个人所有资源，是用户日常工作的主操作页面。

**提示词：**

```
设计一个个人媒体素材库管理页面。

页面整体布局为经典的左右结构：
- 左侧栏目导航树（宽度约220px）：
  - 顶部显示"个人库"标题。
  - 无栏目树结构，个人库为扁平化管理。
  - 底部显示存储用量进度条，格式为"已用 XX GB / 总计 XX GB"。

- 右侧主内容区：
  - 顶部工具栏（从左到右）：
    - 搜索栏：包含搜索输入框，右侧有下拉切换（意图检索/全部/标题/标签）。
    - 资源类型筛选标签栏：全部 | 视频 | 音频 | 图片 | 文档 | 文稿，选中态为蓝色高亮。
    - 排序下拉：创建时间/文件名称/文件大小，旁边有正序/逆序切换图标。
    - 高级检索按钮（漏斗图标）。
    - 视图切换按钮：卡片模式/列表模式。
    - 文件信息面板切换按钮（侧边栏图标）。
    - 上传按钮（蓝色主按钮，带下拉菜单：文件上传/文件夹上传/百度网盘上传）。
  
  - 资源列表区域：
    - 卡片模式（默认）：网格布局，每行4-5个卡片，每个卡片包含：
      - 封面图区域（16:9比例），左上角显示资源类型图标，右下角显示时长（视频/音频）。
      - 视频资源鼠标悬浮时，左右移动可快速预览视频帧。
      - 封面图上方覆盖状态标识：AI状态角标、版本号角标。
      - 卡片底部：资源名称（单行截断）、创建人、创建时间。
      - 悬浮时出现：复选框（左上角）、复制标题按钮、更多操作按钮（三点图标）。
    - 列表模式：标准表格，列包括复选框、封面缩略图、资源名称、类型、创建人、创建时间、大小、状态、操作。
  
  - 底部分页栏：显示总数、每页条数切换（10/20/50/100）、页码导航、跳转输入框。

右键菜单/更多操作菜单项：
- 分享、下载、重命名、公开/不公开、智能处理、入成品库、工具生产、复制文本链接、关联版本、删除。

选中资源后顶部出现批量操作栏：
- 已选X项 | 全选 | 取消全选 | 删除 | 分享 | 入成品库 | 智能处理。
```

## 3 公共库页面

**页面定位：** 汇聚所有用户公开的资源，提供只读浏览、下载和复制链接功能，不支持编辑和删除操作。

**提示词：**

```
设计一个公共资源浏览页面，用于展示平台所有用户公开的媒体资源。

布局与个人库页面一致（左右结构），但有以下关键区别：
- 左侧导航树顶部标题为"公共库"。
- 没有上传按钮。
- 资源卡片和列表项上不显示复选框（不支持批量操作）。
- 右键菜单/操作菜单仅包含：下载、复制文本链接。
- 不显示批量操作栏。
- 卡片上不展示编辑相关的状态标识（如编目状态、审核状态）。

其余搜索、筛选、排序、分页功能与个人库保持一致。

交互说明：
- 点击资源卡片进入只读版资源详情页。
- 下载操作：视频/音频弹出码率选择弹窗，图片直接下载。
- 复制文本链接：弹出弹窗展示多个码率对应的播放地址和下载地址，支持一键复制。
```

## 4 成品库页面

**页面定位：** 组织级资源管理空间，通过多级栏目树组织资源，支持权限控制、工作流、转码组绑定，是团队协作管理媒资的核心页面。

**提示词：**

```
设计一个组织级媒体资源管理页面，即"成品库"。

页面整体布局为左右结构：

- 左侧栏目树导航（宽度约240px）：
  - 顶部："成品库"标题 + 搜索栏目输入框。
  - 栏目树：多级树形结构，支持展开/收起。
    - 每个栏目节点显示栏目名称和资源数量。
    - 鼠标悬浮栏目时出现操作按钮组：新建子栏目(+) | 更多操作(⋯)。
    - 更多操作下拉菜单：重命名、收藏、排序、设置（绑定工作流/转码组）、权限设置、绑定编目模板、专辑编目、删除。
    - 选中栏目高亮显示，右侧内容区展示该栏目下的资源。
  - 栏目树下方显示"栏目收藏夹"区域，展示已收藏的栏目快捷入口。
  - 底部显示存储用量进度条。

- 右侧主内容区（与个人库结构一致，但操作菜单不同）：
  - 顶部工具栏：搜索栏 + 类型筛选 + 排序 + 高级检索 + 视图切换 + 信息面板 + 上传按钮。
  - 资源列表区域：卡片模式/列表模式，卡片上额外显示审核状态、CDN状态、CMS状态标识。
  - 底部分页栏。

右键菜单/更多操作菜单项（成品库资源）：
- 分享、下载、重命名、智能处理、提交审核、CDN分发、推送CMS、移动、收藏、共享、工具生产、复制文本链接、关联版本、补码、删除。

选中资源后顶部出现批量操作栏：
- 已选X项 | 全选 | 取消全选 | 删除 | 分享 | 移动 | CDN分发 | 推送CMS | 提交审核 | 智能处理 | 编目。

交互说明：
- 栏目树支持拖拽排序（同级栏目间）。
- 新建栏目时弹出输入框，输入名称点击确定即创建。
- 删除栏目：仅空栏目可删除，非空时提示"该栏目下有子栏目或资源，无法删除"。
- 上传资源时自动进入当前选中栏目，并使用该栏目绑定的转码组。
```

## 5 版权库页面

**页面定位：** 统一管理平台所有版权资源，展示版权权利项详情，提供到期提醒和版权确认机制。

**提示词：**

```
设计一个版权资源管理页面。

页面布局：
- 顶部工具栏：
  - 搜索栏：支持按版权名称搜索。
  - 筛选项：版权类型下拉、所属栏目下拉、版权状态下拉（全部/有效/即将到期/已到期）。
  - 排序：到期时间/创建时间。

- 资源列表区域（表格模式为主）：
  - 列字段：封面缩略图、资源名称、版权名称、版权类型、权利项摘要、使用地域、使用渠道、有效期起止时间、版权状态、操作。
  - 版权状态列使用颜色标识：
    - 绿色"有效"：正常使用期内。
    - 橙色"即将到期"：30天内到期，显示剩余天数。
    - 红色"已到期"：已过期。
  - 操作列：查看详情、下载、查看下载记录。

- 底部分页栏。

交互说明：
- 点击"查看详情"进入资源详情页，自动定位到版权信息Tab。
- 点击"下载"：弹出版权使用限制确认弹窗，内容包括：
  - 版权权利项摘要。
  - 使用限制说明（地域、渠道、时间）。
  - "我已知晓版权限制条款"复选框。
  - 确认下载 / 取消 按钮。
  - 用户勾选确认后方可下载。
- 点击"查看下载记录"弹出弹窗，展示该资源的下载历史（下载人、时间、IP）。
```

## 6 知识库页面

**页面定位：** 将资源按主题进行知识关联和汇聚，包括系统预置的公共知识库和用户自建的知识库。

**提示词：**

```
设计一个知识库管理页面，分为公共知识库和我的知识库两个区域。

页面布局：
- 左侧导航（宽度约220px）：
  - "公共知识库"分组标题，下方列出10个预置知识库：
    人物库、事件库、节日节气、场景汇聚、重要讲话、敏感库、纪念日、专家库、专家讲座、机构库。
    每个知识库项显示名称和资源数量。
  - 分隔线。
  - "我的知识库"分组标题 + 新建按钮(+)。
    下方列出用户自建的知识库，每项显示名称和资源数量。
    悬浮出现编辑(✏️)和删除(🗑️)按钮。

- 右侧内容区：
  - 选中某个知识库后，展示该知识库下汇聚的资源列表。
  - 顶部显示知识库名称和描述。
  - 资源列表支持卡片/列表模式切换、搜索、筛选、排序。
  - 公共知识库资源为只读，我的知识库资源支持移除。

交互说明：
- 点击"新建"按钮弹出知识库创建弹窗（见独立提示词#29）。
- 公共知识库的资源由系统自动汇聚，不可手动编辑。
- 我的知识库根据用户设置的筛选条件自动从素材库中汇聚资源。
```

## 7 共享库页面

**页面定位：** 管理资源的定向共享，分为"共享给我"和"我的共享"两个视角。

**提示词：**

```
设计一个资源共享管理页面，包含两个Tab切换视图。

页面布局：
- 顶部Tab栏：共享给我 | 我的共享。

【共享给我 Tab】
- 展示其他用户共享给当前用户的资源列表。
- 列表字段：封面图、资源名称、共享人、共享时间、到期时间、共享状态（有效/已过期）、操作。
- 操作列：查看、取用、下载。
- 支持搜索和筛选（按共享人、状态、时间范围）。

交互说明：
- "查看"：进入资源详情页（只读）。
- "取用"：弹出栏目选择弹窗，用户选择目标栏目后，系统将资源复制一份到该栏目。
- "下载"：多码率选择下载。
- 已过期的共享资源仅支持查看，不支持取用和下载。

【我的共享 Tab】
- 展示当前用户已共享出去的资源列表。
- 列表字段：封面图、资源名称、共享范围摘要、共享时间、到期时间、取用次数、操作。
- 操作列：编辑共享、取消共享、查看取用记录。

交互说明：
- "编辑共享"：弹出共享设置弹窗（见独立提示词#24），可修改共享范围和到期时间。
- "取消共享"：二次确认后取消共享，已被取用的资源不受影响。
- "查看取用记录"：弹窗展示取用历史（取用人、取用时间、目标栏目）。
```

## 8 回收站页面

**页面定位：** 展示用户已删除的资源，支持还原或彻底删除。

**提示词：**

```
设计一个回收站页面，展示已删除的媒体资源。

页面布局：
- 顶部工具栏：
  - 搜索栏。
  - "清空回收站"按钮（红色警告按钮，在右侧）。
- 资源列表（表格模式）：
  - 列字段：复选框、封面缩略图、资源名称、原所属栏目、资源类型、删除人、删除时间、操作。
  - 操作列：还原、彻底删除。
- 底部分页栏。

交互说明：
- "还原"：资源恢复到原栏目。如原栏目已删除，弹出栏目选择弹窗让用户选择新栏目。
- "彻底删除"：二次确认弹窗——"该操作不可恢复，确定要彻底删除吗？"确认后物理删除。
- "清空回收站"：二次确认弹窗——"确定要清空回收站中的所有资源吗？此操作不可恢复。"
- 支持批量选择后批量还原或批量彻底删除。

注意：
- 转码失败的资源在列表页删除时直接物理删除，不进入回收站。
```

## 9 资源上传弹窗

**页面定位：** 资源上传入口，支持文件上传、文件夹上传和百度网盘上传三种方式。

**提示词：**

```
设计一个资源上传弹窗，宽度约700px。

弹窗布局：
- 顶部Tab栏：文件上传 | 文件夹上传 | 百度网盘上传。

【文件上传 Tab】
- 顶部拖拽上传区域：虚线框，中间显示上传图标和"点击或拖拽文件到此区域"提示文字。
- 拖拽区域下方为上传配置区：
  - 视频转码组：下拉选择（可选项来自系统配置的转码组列表）。
  - 音频转码组：下拉选择。
  - 密级设置：下拉选择（公开/秘密/不公开/绝密/机密）。
  - 开启去重：开关按钮，开启后上传前检测已存在的同名文件并提示。
- 文件列表区域（选择文件后出现）：
  - 每行显示：文件名（可点击编辑重命名）、文件大小、上传进度条、转码进度条、状态标识、删除按钮(×)。
  - 状态标识：等待中/上传中(百分比)/上传完成/转码中(百分比)/完成/失败。
```

## 29 知识库创建/编辑弹窗

**页面定位：** 用户创建或编辑自定义知识库的弹窗，设置筛选条件自动汇聚资源。

**提示词：**

```
设计一个知识库创建/编辑弹窗，宽度约600px。

弹窗内容：
- 顶部标题："新建知识库"（编辑时为"编辑知识库 - [知识库名称]"），右侧关闭按钮。

- 基本信息区域：
  - 知识库名称：文本输入框（必填）。
  - 描述：多行文本框（选填）。

- 资源筛选条件区域：
  - 标题："自动汇聚条件"，说明："满足以下条件的资源将自动汇聚到此知识库。"
  - 条件组构建器（规则引擎式UI）：
    - 每行一个条件：字段下拉 + 运算符下拉 + 值输入 + 删除按钮。
    - 可选字段：名称、文件大小、标签、创建日期、资源类型、宽度、高度、时长。
    - 运算符根据字段类型动态变化：
      - 文本类：包含/不包含/等于/不等于。
      - 数值类：等于/大于/小于/范围。
      - 日期类：等于/之前/之后/范围。
      - 枚举类：包含/不包含。
    - 条件间关系：AND（所有条件都满足）/ OR（任一条件满足）切换。
    - "+添加条件"按钮。

  - 汇聚范围说明："将从您有权限访问的所有素材中自动汇聚。"

- 预览区域（可折叠）：
  - "预览匹配结果"按钮，点击后展示符合条件的资源数量和前几条结果预览。

- 底部按钮：保存（蓝色主按钮）| 取消。

交互说明：
- 保存后系统按条件自动汇聚资源，后续新增资源如果符合条件也自动纳入。
- 修改条件后重新汇聚。
- 至少设置一个条件才能保存。
```
