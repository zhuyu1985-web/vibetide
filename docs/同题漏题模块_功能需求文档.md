## 一、模块概述

### 1.1 业务背景

融媒体编辑部在日常新闻生产中面临两个核心痛点：

**痛点一：同题报道缺乏系统性对标分析。** 同一新闻事件，央级媒体（人民日报/新华社/央视）、省级媒体（各省党报/都市报/澎湃等）、本地媒体的报道角度、深度、时效各有差异。编辑部缺少工具快速了解"别人怎么报的、我们报得怎么样、差距在哪里"。

**痛点二：热点话题漏报发现滞后。** 全国各大媒体每天产出海量内容，社交媒体热点瞬息万变。编辑部依赖人工盯盘，效率低、覆盖窄，往往错过重要话题的最佳报道窗口。

### 1.2 模块定位

同题漏题模块作为创作者中心的核心分析工具，服务于编辑部的**选题决策**和**报道质量复盘**两大场景，通过AI能力实现：

- **同题对比**：以"我方已发布作品"为起点，自动聚合全网同题报道，AI提炼各方报道视角差异，辅助编辑部进行报道质量复盘与策略优化
- **漏题筛查**：以舆情数据、社媒热榜、竞品媒体动态为信号源，自动比对我方发布记录，筛查疑似漏题，AI生成补报建议

### 1.3 用户角色

| 角色 | 使用场景 | 核心诉求 |
|------|---------|---------|
| 编辑部主任/总编 | 每日选题会、周度复盘 | 快速掌握报道覆盖度和质量差距 |
| 频道/栏目负责人 | 日常内容管理 | 了解本频道同题表现、及时发现漏题 |
| 一线编辑/记者 | 稿件撰写前的调研 | 查看全网同题报道、获取报道角度建议 |

### 2.2 自有作品管理

#### 2.2.1 功能描述

支持对接发布系统，汇聚展示我方已发布的作品信息，作为同题对比的"基准锚点"。每一篇自有作品可作为一个"同题"的起点，触发全网报道检索和对比分析。

#### 2.2.2 页面结构

**页面入口：** 创作者中心 → 同题漏题 → 同题对比（默认Tab）

**顶部筛选区：**

| 筛选项 | 类型 | 说明 |
|-------|------|------|
| 时间范围 | 日期区间选择器 | 默认近7天，支持自定义范围，最长90天 |
| 发布渠道 | 多选下拉 | 数据来源于发布系统的渠道配置（如：APP/微信公众号/微博/抖音/视频号/网站等） |
| 关键词搜索 | 文本输入 | 支持按标题关键词模糊搜索 |
| 同题状态 | 单选标签 | 全部 / 已生成同题分析 / 未生成 |

**作品列表区（表格形式）：**

| 列名 | 说明 | 交互 |
|------|------|------|
| 作品标题 | 稿件标题，超长截断显示 | 点击进入同题对比详情页 |
| 发布时间 | 首次发布时间 | 支持排序 |
| 发布渠道 | 展示所有渠道标签，最多展示3个+「更多」 | hover展示全部 |
| 内容类型 | 图文 / 视频 / 直播回放 / 短视频 | 图标+文字 |
| 传播数据 | 阅读量/播放量、点赞、评论、转发 | 数值展示，支持按阅读量排序 |
| 同题报道数 | 全网匹配到的同题报道文章数量 | 数字Badge样式 |
| 操作 | 「查看同题对比」按钮 | 点击进入详情 |

#### 2.2.3 数据对接要求

- **发布系统对接：** 通过API对接现有发布系统（星云/潮汐发布模块），定时拉取（建议频率：每30分钟）作品基础信息
- **传播数据对接：** 对接各渠道数据回流接口，获取阅读量、点赞、评论、转发等传播指标；不同渠道的指标名称需做归一化映射
- **同题匹配：** 基于作品标题+正文提取关键主题词，调用舆情系统/全网检索接口进行同题匹配，匹配结果缓存并定时更新（建议频率：每2小时）

#### 2.2.4 交互细节

- 列表默认按发布时间倒序排列
- 每页展示20条，支持翻页
- 作品标题列支持inline预览（hover显示前200字摘要）
- 传播数据使用数值缩写（如 1.2万、35.6万）
- 新发布24小时内的作品标记"新"标签

### 2.3 全网报道概览（AI总结）

#### 2.3.1 功能描述

从自有作品列表点击进入后，展示该主题在全网的报道态势概览。核心是**AI总结面板**，由大模型对聚合到的全网报道进行多维度分析总结。

#### 2.3.2 页面结构

**页面路径：** 同题对比 → 点击具体作品 → 全网报道概览

**顶部信息栏：**
- 我方作品标题（加粗大字）
- 发布时间 | 发布渠道标签组 | 我方传播数据摘要
- 「刷新数据」按钮（手动触发最新全网检索）
- 「返回列表」面包屑导航

**数据概览卡片组（横向排列）：**

| 卡片 | 内容 | 样式 |
|------|------|------|
| 全网报道总量 | 匹配到的报道文章总数 | 大数字 + 趋势箭头（与上次刷新对比） |
| 央级媒体报道数 | 人民日报/新华社/央视等央级媒体的报道数量 | 红色强调 |
| 省级媒体报道数 | 省级党报/都市报等的报道数量 | 蓝色强调 |
| 其他媒体报道数 | 自媒体/行业媒体/地方媒体等 | 灰色 |
| 报道时间跨度 | 最早报道时间 ~ 最新报道时间 | 文字描述 |

**AI总结面板：**

该面板为本页核心功能区，分为四个板块，每个板块独立呈现：

**板块一：官媒及央媒报道分析**
- 列出已检索到的央级媒体（人民日报、新华社、央视新闻等）的报道标题
- AI提炼央级媒体的**报道角度/切入点**（如：政策解读视角、民生影响视角、国际对比视角等）
- AI总结央级媒体报道的**关键论点**和**引用的权威信源**
- 每条来源支持「查看原文」链接跳转

**板块二：其他媒体报道分析**
- 按省级媒体、都市媒体、行业媒体、自媒体分类展示
- AI提炼各层级媒体的**差异化报道视角**
- AI分析不同媒体在**深度、时效、独家性**上的表现差异
- 每条来源支持「查看原文」链接跳转

**板块三：报道亮点与创新点**
- AI从全网报道中提炼出**最具创新性的报道角度**（如独家数据、独特视角、新型表达形式等）
- AI标注哪些角度是**我方报道未涉及的**
- AI给出**可借鉴的报道策略建议**

**板块四：整体报道总结**
- AI对该主题的全网报道态势做**一段话总结**（200-300字）
- 包含：报道热度走势判断、主流舆论方向、我方报道在全网中的定位评估
- 给出**我方后续跟进建议**（是否需要追加报道、建议什么角度）

- 来源原文链接在新窗口/新Tab打开
- AI总结面板支持**折叠/展开**各板块
- 首次进入时自动触发AI分析，后续进入使用缓存结果（缓存有效期2小时），显示"上次分析时间"

### 2.4 全网报道文章列表

#### 2.4.1 功能描述

展示同题下全网检索到的所有报道文章明细，支持快速浏览和AI解读。

#### 2.4.2 页面结构

**位置：** 全网报道概览页面的下方Tab，或作为独立子Tab页

**筛选与排序区：**

| 筛选项 | 类型 | 说明 |
|-------|------|------|
| 媒体级别 | 多选标签 | 央级 / 省级 / 市级 / 行业 / 自媒体 |
| 时间排序 | 切换按钮 | 最新优先 / 最早优先 |
| 关键词过滤 | 文本输入 | 在当前同题报道中二次检索 |
| 内容类型 | 多选 | 图文 / 视频 / 短视频 / 直播 |

**文章列表（卡片式或列表式切换）：**

**列表模式字段：**

| 列名 | 说明 | 交互 |
|------|------|------|
| 报道标题 | 文章标题 | 点击打开原文（新窗口） |
| 来源媒体 | 媒体名称 + 级别标签（央/省/市/行业/自媒体） | 级别标签使用不同颜色区分 |
| 报道时间 | 发布时间 | — |
| 作者/账号 | 报道记者或发布账号名称 | — |
| 报道摘要 | 正文前150字预览 | hover展示更多 |
| AI解读 | 「AI解读」按钮 | 点击展开/弹窗显示解读结果 |

**卡片模式：**
- 每张卡片展示：标题（加粗）、来源媒体（级别标签）、发布时间、摘要前两行
- 卡片右上角：「查看原文」图标按钮
- 卡片底部：「AI解读」按钮

#### 2.4.3 AI解读功能

点击「AI解读」后，对单篇文章进行深度解读，结果以抽屉面板/弹窗形式展示。

**单篇文章AI解读 Prompt：**

```
你是一位资深媒体内容分析师。请对以下新闻报道进行深度解读：

【文章信息】
标题：{article_title}
来源：{source_outlet}（{media_level}级媒体）
发布时间：{publish_time}
正文内容：{article_content}

【对比基准（我方报道）】
我方标题：{our_title}
我方发布时间：{our_publish_time}
我方报道摘要：{our_content_summary}

请输出以下解读内容（JSON格式）：

{
  "core_angle": "该报道的核心报道角度（30字以内）",
  "key_information": ["该报道中的关键信息点（3-5条）"],
  "unique_content": "与我方报道相比，该文独有的信息/视角（50字以内）",
  "writing_technique": "该报道的写作手法/叙事特点（30字以内）",
  "source_analysis": "该报道引用的信源分析（如专家/官方/数据等）",
  "reference_value": {
    "level": "high/medium/low",
    "reason": "对我方的参考价值说明（50字以内）"
  }
}
```

#### 2.4.4 交互细节

- AI解读结果缓存，同一文章不重复调用
- 支持批量选择文章后「批量AI解读」（一次最多5篇）
- 列表/卡片视图切换状态持久化（记住用户偏好）
- 文章列表支持无限滚动加载或翻页（每页30条）

### 2.5 竞品媒体对标

#### 2.5.1 功能描述

定向查看预设的竞品媒体单位在该同题下的发布作品，按媒体级别分类标识，便于编辑部进行精准对标分析。

#### 2.5.2 前置配置：竞品媒体库

**配置入口：** 系统设置 → 同题对比设置 → 竞品媒体管理

| 配置项 | 说明 |
|-------|------|
| 媒体名称 | 竞品媒体单位名称 |
| 媒体级别 | 央级 / 省级 / 市级 / 区县级 / 行业 |
| 媒体标识 | 系统根据级别自动生成标识样式（如：央级=红色、省级=蓝色、市级=绿色） |
| 监测账号 | 该媒体在各平台的官方账号（微信公众号ID、微博UID、抖音号等） |
| 优先级 | 高/中/低（高优先级媒体会在对标分析中优先展示和分析） |
| 状态 | 启用/停用 |

**预置模板：**
- 系统预置全国主流央级媒体名单（人民日报、新华社、央视新闻、光明日报、经济日报等）
- 系统预置全国省级党报名单
- 支持一键导入/按区域批量添加

#### 2.5.3 页面结构

**位置：** 同题对比详情页的子Tab「竞品对标」

**展示方式：按媒体级别分组展示**

**分组一：央级媒体**（红色分组标识）

| 列名 | 说明 |
|------|------|
| 媒体名称 | 媒体单位名称 + 级别标识Badge |
| 报道作品标题 | 该媒体在此同题下发布的作品标题；若有多篇则展示列表 |
| 报道主体 | 该报道聚焦的主体/对象（AI提取） |
| 发布时间 | 作品发布时间 |
| 发布渠道 | 在哪个平台发布（微信/微博/APP等） |
| AI解读 | 「AI解读」按钮 |
| 查看原文 | 跳转链接 |

**分组二：省级媒体**（蓝色分组标识）
- 同上结构

**分组三：市级/其他媒体**（绿色/灰色分组标识）
- 同上结构

**空状态处理：**
- 若某级别媒体在该同题下无报道，显示"该级别竞品媒体暂无同题报道"
- 并标注该结果可能的原因（如：该事件地域性较强，央级媒体关注度有限）

## 三、漏题筛查（Missing Topic Detection）

### 3.1 功能总览

```
漏题筛查

### 3.3 疑似漏题智能识别

#### 3.3.1 功能描述

系统自动将线索库中的线索与我方已发布作品库进行比对，识别出我方未报道的热点话题/事件/竞媒作品，标记为"疑似漏题"。

#### 3.3.2 匹配算法逻辑

```
线索库中的每条线索
    ↓
提取线索的关键主题词/事件实体
    ↓
在我方作品库中检索（标题+正文语义匹配）
    ↓
匹配度 > 阈值A（如0.75） → 标记为「已覆盖」
匹配度 在 阈值B ~ 阈值A（如0.45~0.75） → 标记为「待人工确认」
匹配度 < 阈值B（如0.45） → 标记为「疑似漏题」
    ↓
疑似漏题按照 [热度值 × 时效衰减系数] 排序
```

**时效衰减系数：**
- 1小时内的线索：系数1.0
- 1-6小时：系数0.8
- 6-24小时：系数0.5
- 24小时以上：系数0.3

#### 3.3.3 疑似漏题列表页面

**位置：** 漏题筛查页面 → 筛选「疑似漏题」状态 或 独立的「漏题预警」子Tab

**展示方式：卡片式列表，按紧急程度排列**

每张漏题卡片包含：

| 区域 | 内容 |
|------|------|
| 标题区 | 线索标题（加粗）+ 漏题紧急程度标签（🔴紧急/🟡一般/🟢关注） |
| 来源标签区 | 线索来源标识（如：「微博热搜#5」「人民日报」「舆情预警」） |
| 摘要区 | 线索内容摘要（150字以内） |
| 热度指标 | 热度值进度条 + 数值 |
| 时效指标 | 距发现时间（如：1小时前发现） |
| 操作按钮组 | 「查看详情」「AI分析」「排除」「手动查证」 |

**紧急程度判定规则：**
- 🔴紧急：热度值>80 且 央级媒体已报道 且 发现时间<6小时
- 🟡一般：热度值50-80 或 省级媒体已报道
- 🟢关注：热度值<50 且 仅自媒体/社媒渠道

### 3.4 漏题详情与AI分析

#### 3.4.1 功能描述

查看漏题线索的原文详情，并支持一键AI分析全网报道情况，生成与"同题对比"中相同结构的全网报道总结。

#### 3.4.2 页面结构

**页面路径：** 漏题筛查 → 疑似漏题列表 → 点击「查看详情」

**左侧：漏题原文面板**
- 线索标题
- 来源信息（来源媒体/平台 + 原始链接）
- 来源标识标签（社媒热榜/对标媒体/舆情热点）
- 原文内容展示（如为外链，则嵌入iframe或展示抓取的正文）
- 发布时间
- 热度指标

**右侧：AI分析面板**

**操作按钮：** 「一键AI检索全网报道」

点击后，系统执行以下流程：
1. 以漏题线索为主题词，调用全网检索接口获取同题报道列表
2. 将报道列表输入大模型，生成全网报道情况总结
3. 流式输出分析结果

**AI总结内容结构（与同题对比2.3一致）：**
- 板块一：官媒及央媒报道分析
- 板块二：其他媒体报道分析
- 板块三：报道亮点与创新点
- 板块四：整体报道总结

**额外增加的漏题专属板块：**

**板块五：补充报道建议**

### 3.7 漏题推送与处置

#### 3.7.1 功能描述

#### 3.7.2 推送目标配置

| 推送通道 | 对接方式 | 推送内容 |

| 采编系统（星云/潮汐） | 内部API | 创建待采任务或线索条目 |
| 站内消息 | 系统消息 | 推送至指定用户的消息中心 |

#### 3.7.3 推送交互流程

**步骤一：** 在漏题详情页或疑似漏题列表页，点击「推送处置」按钮

|-------|------|

| 推送对象 | 选择接收人/接收群组（从通讯录选择） |
| 附加信息 | 编辑推送消息内容（系统预填模板，支持修改） |
| 紧急程度 | 普通/紧急/特急 |
| 附带AI分析 | 开关，是否将AI生成的补充报道建议一并推送 |

**推送消息模板（预填）：**

```
【漏题预警】{urgency_label}
话题：{topic_title}
发现来源：{source_label}
当前热度：{heat_score}/100

- 央级媒体：{central_status}

**步骤三：** 确认推送

- 点击「确认推送」后执行
- 推送状态反馈：成功/失败/部分成功
- 推送成功后，该漏题状态变更为「已推送」，记录推送时间、推送通道、接收人

#### 3.7.4 推送记录

**位置：** 漏题详情页底部「处置记录」区域

| 字段 | 说明 |
|------|------|
| 推送时间 | 操作时间 |
| 推送通道 | 推送到哪个系统/通道 |
| 推送人 | 操作者 |
| 接收人 | 消息接收者 |

### 4.2 AI能力通用规范

| 规范项 | 要求 |
|-------|------|
| 模型调用 | 统一通过AI网关调用，支持模型路由（总结类任务用长文本模型，解读类用指令模型） |
| 流式输出 | 所有AI总结/解读功能必须支持流式输出（SSE） |
| 结果缓存 | AI分析结果缓存2小时，过期后可手动刷新，缓存Key = 主题哈希 + 报道数据快照哈希 |
| 重试机制 | AI调用失败时自动重试1次，仍失败则展示"分析暂时不可用，请稍后重试"提示 |
| Token限制 | 单次总结输入控制在8000 token以内，超出时按优先级截断（央级 > 省级 > 其他） |
| 结果格式 | 全部使用JSON格式化输出，前端根据JSON结构渲染UI |
| 反馈机制 | 每次AI输出底部增加「有帮助👍」/「需改进👎」反馈按钮，用于后续prompt优化 |
